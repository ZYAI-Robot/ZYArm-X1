# 贡献指南

感谢你帮助改进 ZYArmV1。本项目希望服务于学生、教师、创客、研究者，以及基于真实机械臂硬件做二次开发的开发者。

## 贡献授权

提交贡献即表示你同意：你的贡献按被修改文件或目录适用的协议授权。

| 贡献范围 | 贡献适用协议 |
| --- | --- |
| `firmware/` 下项目原创固件源码 | MPL-2.0 |
| SDK、ROS 2 包、Web 应用、示例、工具、诊断脚本、构建脚本，以及 `firmware/` 之外的其他软件源码 | Apache-2.0 |
| URDF、xacro、launch 文件、配置文件和运动学机器人描述文件 | Apache-2.0 |
| 文档和脱敏后的公开 mesh 资产 | CC-BY-SA-4.0 |

如果你的贡献包含第三方材料、生成文件、厂商文件或其他许可证下的文件，请在提交说明或 pull request 中明确说明来源和许可证。

## 不应贡献的内容

请勿向公开仓库贡献：

- 生产 CAD、高精生产 mesh、BOM 或制造工艺文件。
- 工厂标定数据、产测流程、私有密钥、签名密钥、客户数据或设备密钥。
- 未确认授权的专有 SDK、商业课程、书籍、网站或其他来源中的文件。
- 可能使用户误认为官方 ZYArm 所有或官方背书的品牌资产、Logo 或产品标识。

## 固件安全

固件变更可能驱动真实硬件运动。提交固件相关变更时，请特别注意：

- 保留停止、复位、限位和状态上报行为，除非变更目标明确要求修改这些契约。
- 记录会影响关节运动、速度限制、扭矩、标定、刷机或恢复流程的行为变化。
- 在建议用户使用前，尽量在合适的硬件或仿真环境中验证固件变更。

修改固件可能影响设备安全、性能、标定和质保。

## 评审说明

维护者可能要求补充署名、许可证说明、测试结果、安全说明或贡献者协议，尤其是涉及固件、公开 mesh、机器人描述资源或大规模外部贡献时。
