# 快速上手

本章面向第一次接触 ZYArmV1 / `ZYArm-X1` 的用户，目标是只用串口软件直接发送固件指令，完成一次真实机械臂的最小验证闭环。

本章不要求安装 Python SDK、ROS 2、MoveIt、Gazebo、LeRobot 或 Web 项目。

## 先认识这些词

| 术语 | 本章里的含义 |
| --- | --- |
| 串口 | 电脑和控制板之间的文本通信通道 |
| CH340 | 控制板常用的 USB 转串口芯片，Windows 可能需要驱动 |
| `COMx` / `/dev/ttyUSBx` | Windows / Ubuntu 看到的串口设备名 |
| `230400, 8N1` | 本章使用的串口通信参数 |
| `CMD` | 发送给固件的文本命令格式 |
| `STATUS` | 固件返回的机械臂状态，包含关节和夹爪数值 |

如果某个词看不懂，可以先查 [术语与工具速查](../术语与工具速查.md)。

## 本章主线

```text
准备硬件和串口软件
  -> 确认固定状态、安全空间和断电方式
  -> 找到 COMx / ttyUSBx
  -> 打开串口：230400, 8N1, no flow control
  -> 发送 CMD6 查询状态：[CMD][6]
  -> 确认能看到 [STATUS]
  -> 发送 CMD1 复位到初始姿态：[CMD][1]
  -> 发送 CMD0 IK 小动作：[CMD][0][200 0 0 0 0 0]
  -> 再次发送 CMD6，记录 J0-J5 和 CLAW
  -> 发送 CMD1 复位：[CMD][1]
  -> 发送 CMD3，把刚刚记录的状态发回去
  -> 发送 CMD1 再次复位
```

## 你将得到什么

完成本章后，你应该可以：

- 确认机械臂已经正确上电。
- 找到 Windows `COMx` 或 Ubuntu `/dev/ttyUSBx` 串口。
- 使用串口软件发送固件文本命令。
- 通过 `CMD6` 看到 `[STATUS]` 状态返回。
- 通过 `CMD1` 让机械臂回到初始姿态。
- 通过 `CMD0` 完成一次由固件 IK 控制的小动作。
- 通过 `CMD3` 复现刚刚记录到的关节状态。
- 知道异常时优先直接切断机械臂电源。

## 安全提醒

第一次动作前请确认：

- 机械臂周围没有人手、线材、杯子或其他障碍物。
- 你知道如何快速切断机械臂电源。
- 已经成功读取 `[STATUS]`。
- 如果机械臂姿态明显异常，先不要发送动作命令。

如果状态读取失败，不要继续发送动作命令。动作过程中如果出现异常，优先直接断电；串口已经连通且来得及发送时，也可以发送 `[CMD][2]` 指令急停。
