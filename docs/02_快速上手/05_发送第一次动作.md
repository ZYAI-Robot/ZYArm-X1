# 发送第一次动作

第一次动作只用于确认链路和执行能力，不追求复杂控制。本章使用固件 `CMD0` 运动学逆解命令，让固件自动计算各关节运动。

运动学逆解简称 IK，可以先理解成“告诉机械臂末端想去哪里，由固件计算各个关节该怎么转”。这比第一次就手写 6 个关节角更适合入门验证。

## 动作前检查

- 已经成功读取 `[STATUS]`。
- 机械臂周围留出空间。
- 你可以快速断电。
- 当前姿态适合执行复位动作和快速上手动作。

## 先复位到初始姿态

在串口软件里发送：

```text
[CMD][1]
```

正常情况下，你会看到类似：

```text
ACK_RECEIVED: CMD_ID=1
ACK_COMPLETED: CMD_ID=1, SUCCESS
```

复位完成后，再执行下面的 IK 小动作。如果复位过程中出现异常，请直接断电。

## 再发送 IK 小动作

在串口软件里发送：

```text
[CMD][0][200 0 0 0 0 0]
```

命令定义位置：

- [firmware/Core/Src/arm_shell_cmd.c](../../firmware/Core/Src/arm_shell_cmd.c)

这条命令表示发送一个 IK 目标位姿：

```text
x=200, y=0, z=0, rx=0, ry=0, rz=0
```

固件会根据目标位姿计算关节角度，并控制机械臂运动。这里的 `200` 是目标位姿中的 X 方向位置，不是让当前姿态相对移动 200。

`x y z` 表示末端目标位置，`rx ry rz` 表示末端目标姿态。本章只使用项目方确认过的简单示例，不要求你现在理解完整坐标系。

## 关于坐标系

本节只使用一个已经确认过的简单 IK 示例，帮助你验证机械臂可以由固件完成运动学逆解。完整坐标系定义、单位、末端姿态 `rx/ry/rz` 含义和不同型号的适配说明，会在后续基础操作或开发文档中单独展开。

> 后续可补充该 IK 示例适用的出厂初始姿态、单位说明、推荐演示空间和不同型号目标位姿差异。

## 成功标志

正常情况下，你会看到类似：

```text
ACK_RECEIVED: CMD_ID=0
ACK_COMPLETED: CMD_ID=0, SUCCESS
```

并观察到机械臂完成一次小动作。

如果返回 `ERROR`、没有返回或动作异常，请直接切断机械臂电源。确认原因前，不要继续发送动作命令。
