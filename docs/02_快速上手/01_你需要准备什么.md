# 你需要准备什么

## 必备

- ZYArmV1 / `ZYArm-X1` 机械臂本体
- 已安装在机械臂上的控制板，并刷入与当前仓库匹配的固件
- 配套电源，优先使用套件内电源
- USB 串口线，或确认支持数据传输的 USB 线
- 一台电脑，第一版文档默认 Windows 优先
- 串口调试工具
- 金属底板、固定螺丝和桌面固定夹具
- 螺丝刀或电动螺丝刀

## 串口软件要求

快速上手只需要能打开串口并发送文本命令。你可以选择 PuTTY、MobaXterm、Arduino Serial Monitor、SSCOM、XCOM 或其他同类工具。

串口软件的作用很简单：它把你输入的 `[CMD]` 文本发送给控制板，再把固件返回的 `[STATUS]`、`ACK` 等文本显示出来。它不是机械臂专用软件，只是一个通用通信工具。

串口工具需要支持：

- 选择串口号，例如 Windows `COM3` 或 Ubuntu `/dev/ttyUSB0`。
- 设置波特率为 `230400`。
- 以文本方式发送命令，不使用十六进制发送。
- 发送命令时追加回车或换行。
- 能看到机械臂返回的文本内容。

## 推荐串口工具

### Windows

Windows 推荐使用 [Serial Debug Assistant + TCP/UDP](https://apps.microsoft.com/detail/9nblggh43hdm?launch=true&hl=zh-cn&gl=cn)。后续截图以这个工具为例。

<img src="../assets/Images/串口调试助手.png" alt="串口调试助手界面" width="720">

上图用于确认 Windows 串口调试助手的主界面位置；后续串口参数和命令发送截图会沿用这个工具。

### Ubuntu

Ubuntu 推荐使用 CuteCom。安装并启动后，在界面中选择实际串口，并按下图核对通信参数：

```bash
sudo apt-get update
sudo apt-get install cutecom
cutecom
```

<img src="../assets/Images/cutecom.png" alt="CuteCom 串口设置界面" width="720">

图中 `/dev/ttyUSB0` 只是示例，请替换成你电脑上实际出现的设备名。关键参数需要保持为 `230400`、`8` 数据位、`None` 校验、`1` 停止位，输入模式使用普通文本，不要勾选十六进制输出。

如果当前用户没有串口权限，可以临时使用 `sudo cutecom` 验证；长期使用建议按 [连接电脑并找到串口](03_连接电脑并找到串口.md) 中的说明处理串口权限。

## 本章会用到的固件命令

| 命令                                 | 作用                          |
| ---------------------------------- | --------------------------- |
| `[CMD][6]`                         | 查询当前状态                      |
| `[CMD][1]`                         | 复位到初始姿态                     |
| `[CMD][0][200 0 0 0 0 0]`          | 发送一次 IK 小动作                 |
| `[CMD][3][J0 J1 J2 J3 J4 J5 CLAW]` | 复现刚刚记录的状态                   |
| `[CMD][2]`                         | 指令急停，串口已连通且来得及时使用；异常时优先直接断电 |

`CMD` 是固件命令编号，`STATUS` 是固件返回的机械臂状态。第一次上手只需要按表格顺序使用这些命令，不需要先理解所有固件指令。

完整环境矩阵见 [安装与准备](../03_安装与准备/README.md)。

## 本章不要求

- 不要求安装 Python
- 不要求安装 Python SDK
- 不要求安装 ROS 2
- 不要求安装 MoveIt
- 不要求安装 Gazebo
- 不要求安装 LeRobot
- 不要求先启动 Web 项目

这些内容都属于后续路线。先用串口软件把真实机械臂链路跑通，会更容易判断问题发生在硬件、串口、固件还是上层软件。
