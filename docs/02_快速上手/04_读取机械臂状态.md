# 读取机械臂状态

读取状态是第一次控制前最重要的验证步骤。只有能确认状态，才说明电脑、浏览器或串口工具、固件和机械臂之间的链路基本通了。

官方 Web 路线中，状态读取会由页面自动完成；串口备用路线中，你可以手动发送 `CMD6` 查看 `[STATUS]`。

## 官方 Web：确认设备和状态同步

完成上一页的串口选择后，先看页面状态：

- 右上角显示“设备已连接”。
- 页面没有显示设备异常或连接失败。
- 关节控制页里的模型和滑块可以正常显示。

<img src="../assets/Images/web_连接设备成功.png" alt="Web 控制台设备连接成功" width="960">

上图用于确认浏览器已经通过 Web Serial 打开了本地机械臂串口。对第一次使用者来说，只要页面显示设备已连接，并且没有异常提示，就可以继续下一步。

如果页面提示异常，不要继续发送动作。先检查：

- 机械臂是否已经上电。
- USB 数据线是否松动。
- 浏览器是否仍然连接着正确串口。
- 串口是否被其他程序占用。
- Windows 是否能在设备管理器里看到 CH340 对应的 `COMx`。
- 机械臂状态灯是否异常。

## Web 状态对应的底层能力

Web 控制台显示的设备状态，仍然来自固件返回的机械臂状态。串口备用路线里，你可以用 `CMD6` 直接查看同一类底层状态：

```text
[CMD][6]
```

这一步不会让机械臂运动，只是让固件把当前 6 个关节和夹爪状态返回给上位机。

## 串口备用：手动发送 CMD6

当官方 Web 无法连接、状态异常，或你想直接确认底层返回时，可以关闭 Web 页面或断开 Web 连接，改用串口工具打开同一个端口。

发送：

```text
[CMD][6]
```

正常情况下，机械臂会返回类似：

```text
ACK_RECEIVED: CMD_ID=6
[STATUS] J0:... J1:... J2:... J3:... J4:... J5:... CLAW:...
ACK_COMPLETED: CMD_ID=6, SUCCESS
```

<img src="../assets/Images/串口调试助手_CMD6.png" alt="串口调试助手 CMD6 状态返回示例" width="720">

上图用于对照 `CMD6` 的正常返回形式。实际关节角度和夹爪数值会随当前姿态变化，不需要和截图完全一致。

看到 `[STATUS]` 就说明状态读取成功。这里的 `J0` 到 `J5` 是固件返回的六个关节角度，`CLAW` 是夹爪状态。不同机械臂、不同姿态下，具体数值会不同。

`ACK_RECEIVED` 表示固件收到了命令，`ACK_COMPLETED` 表示命令处理结束。第一次上手时重点看是否出现 `[STATUS]`，不需要逐项分析所有 ACK。

## 如果没有状态

如果官方 Web 或串口工具都无法确认状态，不建议继续发送动作命令。按顺序检查：

- 串口号是否选对。
- 机械臂是否已经上电。
- USB 数据线是否支持数据传输。
- CH340 驱动是否安装成功。
- 浏览器或串口工具是否独占了同一个端口。
- 串口参数是否为 `230400, 8N1, no flow control`。
- 串口工具发送命令时是否追加了换行。

仍然失败时，进入 [校准维护与排障](../08_校准维护与排障/README.md)，不要继续发送动作命令。
