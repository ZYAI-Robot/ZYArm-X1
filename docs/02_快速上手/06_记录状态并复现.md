# 记录状态并复现

完成第一次 IK 动作后，本节会再读取一次状态，记录当前关节角度，然后用 `CMD3` 把刚刚记录的状态发回去，验证“状态读取 -> 复位 -> 状态复现”的闭环。

`CMD3` 是多关节同步控制命令。它和 `CMD0` 的区别是：`CMD0` 输入末端位姿，由固件求关节角；`CMD3` 直接输入 6 个关节和夹爪的目标值。

## 1. 再次读取状态

发送：

```text
[CMD][6]
```

正常返回类似。下面数值只是示例，你需要以自己串口软件里实际看到的 `[STATUS]` 为准：

```text
[STATUS] J0:0.20 J1:-102.50 J2:41.70 J3:-0.10 J4:60.70 J5:-0.30 CLAW:0.20
```

把 `J0` 到 `J5` 和 `CLAW` 记录下来。后续 `CMD3` 会直接使用这些数值。

| 字段 | 示例值 | 你的记录值 |
| --- | --- | --- |
| J0 | 0.20 |  |
| J1 | -102.50 |  |
| J2 | 41.70 |  |
| J3 | -0.10 |  |
| J4 | 60.70 |  |
| J5 | -0.30 |  |
| CLAW | 0.20 |  |

## 2. 复位机械臂

发送：

```text
[CMD][1]
```

`CMD1` 会让机械臂回到初始位置。执行复位时同样要保证周围空间安全。

## 3. 复现刚刚记录的状态

`CMD3` 是同步控制六轴和夹爪的命令，格式为：

```text
[CMD][3][J0 J1 J2 J3 J4 J5 CLAW]
```

这里的 `J0` 到 `J5` 和 `CLAW` 是固件角度表达，应直接来自刚刚 `CMD6` 读取到的 `[STATUS]`。第一次上手时，不建议自己编写新的六轴目标。

以上面的示例状态为例，可以发送：

```text
[CMD][3][0.20 -102.50 41.70 -0.10 60.70 -0.30 0.20]
```

如果你的状态返回是：

```text
[STATUS] J0:a J1:b J2:c J3:d J4:e J5:f CLAW:g
```

则发送：

```text
[CMD][3][a b c d e f g]
```

其中 `a` 到 `g` 代表你实际记录到的数值，不是可以直接复制的命令。

## 4. 再次复位

复现完成后，再发送一次：

```text
[CMD][1]
```

机械臂回到初始位置后，本章的最小真机闭环就完成了。

## 异常处理

动作过程中如果出现异常，请直接切断机械臂电源。断电后不要继续复现动作，先进入 [校准维护与排障](../08_校准维护与排障/README.md)。
