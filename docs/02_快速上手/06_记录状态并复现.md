# 记录状态并复现

完成第一次 Web 小幅动作后，本节把刚才的操作整理成一个“可复盘的闭环”。对于第一次使用者来说，最重要的是确认自己已经经历了完整链路：

```text
打开官方 Web
  -> 选择本地机械臂串口
  -> 确认设备已连接和状态同步
  -> 先看模型草稿
  -> 再发送真实动作
  -> 停止或复位回到安全姿态
```

如果你想进一步理解 Web 背后的底层状态，也可以用串口工具执行 `CMD6 -> CMD1 -> CMD3 -> CMD1`，把当前状态读出来、复位、再复现一次。这个串口练习不是官方 Web 快速上手的必经步骤，但很适合排障和进阶学习。

## 1. 记录你的 Web 闭环

建议把第一次成功现象记录下来。课程、社群求助或后续排障时，这张表会比“好像连上了”更有用。

| 项目 | 你的记录 |
| --- | --- |
| 官方 Web 地址 | `https://arm.zyairobot.com/#/zy` |
| 浏览器 | Chrome / Edge / 其他： |
| 系统 | Windows / Ubuntu / macOS： |
| 机械臂串口 | 例如 `COM5` 或 `/dev/ttyUSB0`： |
| 页面是否显示设备已连接 | 是 / 否 |
| 状态是否持续异常 | 是 / 否 |
| 是否先做了模型草稿 | 是 / 否 |
| 是否点击“发送到机械臂”并完成小幅动作 | 是 / 否 |
| 是否完成停止或复位练习 | 是 / 否 |
| 是否知道如何直接断电 | 是 / 否 |

如果这张表里有“否”，先不要进入更复杂玩法。回到对应页面补齐，再继续。

## 2. 复位并结束第一次上手

在官方 Web 中点击“复位”，让机械臂回到初始姿态。复位完成后，如果你暂时不继续操作，可以断开页面设备连接，并关闭机械臂电源。

如果复位失败、页面状态异常或机械臂动作不符合预期，优先直接断电。断电后不要继续重复发送动作，先进入 [校准维护与排障](../08_校准维护与排障/README.md)。

## 3. 串口进阶：读取状态并用 CMD3 复现

下面是底层状态复现练习。它帮助你理解 Web 的“关节同步”和“状态同步”背后是什么，不是第一次 Web 路线必须完成的步骤。

开始前请先断开 Web 页面里的设备连接，或关闭页面，避免 Web 和串口工具同时占用同一个端口。

### 再次读取状态

发送：

```text
[CMD][6]
```

正常返回类似。下面数值只是示例，你需要以自己串口工具里实际看到的 `[STATUS]` 为准：

```text
[STATUS] J0:0.20 J1:-102.50 J2:41.70 J3:-0.10 J4:60.70 J5:-0.30 CLAW:0.20
```

把 `J0` 到 `J5` 和 `CLAW` 记录下来。后续 `CMD3` 会直接使用这些数值。

| 字段 | 示例值 | 你的记录值 |
| --- | --- | --- |
| J0 | 0.20 |  |
| J1 | -102.50 |  |
| J2 | 41.70 |  |
| J3 | -0.10 |  |
| J4 | 60.70 |  |
| J5 | -0.30 |  |
| CLAW | 0.20 |  |

### 复位机械臂

发送：

```text
[CMD][1]
```

`CMD1` 会让机械臂回到初始位置。执行复位时同样要保证周围空间安全。

### 复现刚刚记录的状态

`CMD3` 是同步控制六轴和夹爪的命令，格式为：

```text
[CMD][3][J0 J1 J2 J3 J4 J5 CLAW]
```

这里的 `J0` 到 `J5` 和 `CLAW` 是固件角度表达，应直接来自刚刚 `CMD6` 读取到的 `[STATUS]`。第一次练习时，不建议自己编写新的六轴目标。

以上面的示例状态为例，可以发送：

```text
[CMD][3][0.20 -102.50 41.70 -0.10 60.70 -0.30 0.20]
```

如果你的状态返回是：

```text
[STATUS] J0:a J1:b J2:c J3:d J4:e J5:f CLAW:g
```

则发送：

```text
[CMD][3][a b c d e f g]
```

其中 `a` 到 `g` 代表你实际记录到的数值，不是可以直接复制的命令。

### 再次复位

复现完成后，再发送一次：

```text
[CMD][1]
```

机械臂回到初始位置后，本章的底层状态复现练习就完成了。

## 异常处理

- Web 页面异常：先点击“停止”，再尝试“复位”；仍异常时直接断电。
- 串口工具没有返回：不要继续发送动作，检查串口号、波特率、换行、驱动和电源。
- `CMD3` 复现动作异常：直接断电，不要继续复现刚才的状态。
- 不确定当前是否安全：直接断电比继续猜测更可靠。
