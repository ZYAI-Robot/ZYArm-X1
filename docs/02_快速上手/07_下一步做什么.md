# 下一步做什么

如果你已经通过官方 Web 控制台完成设备连接、状态确认、模型草稿、小幅真机动作、停止和复位，说明第一次使用的核心链路已经跑通。

## 你已经验证了什么

- 官方 Web 控制台可以打开：`https://arm.zyairobot.com/#/zy`。
- 浏览器可以通过 Web Serial 连接本地机械臂串口。
- 机械臂已经正确上电，并能被电脑识别。
- 页面可以区分模型预览和真机发送。
- 小幅真机动作可以成功执行。
- 停止、复位和直接断电的边界已经练习过。
- 串口工具可以作为官方 Web 不可用或需要底层 ACK 时的备用路线。

如果你还额外完成了串口 `CMD6 -> CMD1 -> CMD3 -> CMD1` 练习，也说明你已经理解了状态读取、复位和关节状态复现的底层闭环。

## 第一条推荐路线：继续玩 Web

建议下一步进入 [常用玩法与项目案例](../05_常用玩法/README.md)。那里会把快速上手中的 Web 操作扩展成更完整的练习：

- [Web 控制台入门](../05_常用玩法/01_Web仿真体验.md)：完整认识关节控制、控制台信息、设置页、停止、复位和硬件重启。
- [动作录制与复现](../05_常用玩法/02_动作录制与复现.md)：用 Web 创建动作名称、开始录制、停止保存、查看列表，再理解底层回放命令。
- [Web 动作编排工作流](../05_常用玩法/07_Web动作编排工作流.md)：把复位、等待、夹爪、关节同步和末端移动组合成可保存的流程。

这条路线最适合第一次使用机械臂的学习者，因为它能持续用页面降低操作门槛，同时逐步引出底层串口和固件概念。

## 学习底层串口指令

如果你想知道 Web 页面背后到底发了什么命令，进入 [基础操作](../04_基础操作/README.md)，学习：

- 串口指令格式与回包。
- 读取状态。
- 关节控制。
- 夹爪控制。
- 设置速度。
- 运动学逆解。
- 动作录制与回放。
- 状态上报和诊断。

这条路线适合做排障、写脚本或理解固件协议。

## 使用 SDK 或写脚本

如果你希望用 Python 或 C++ 编写控制程序，可以进入：

- [SDK 说明](../../software/zyarm_sdk/README.md)
- [Serial API 接入](../06_仿真与框架接入/Serial_API/README.md)

写脚本前，建议先能用官方 Web 或串口工具完成状态读取和复位。这样脚本出问题时，你能判断是代码问题还是硬件链路问题。

## Web 路线的边界

公开文档里的 Web 路线只指向官方在线控制台：

```text
https://arm.zyairobot.com/#/zy
```

后续 Web 玩法都继续使用这个官网入口。官方 Web 暂时不可访问时，建议先切换到串口工具或 SDK 完成状态读取、复位和排障。

Web 使用环境和官网不可用时的备用路线见 [Web 控制台使用环境](../03_安装与准备/02_Web仿真环境.md)。

## 遇到问题

进入 [校准维护与排障](../08_校准维护与排障/README.md)，按现象排查串口、动作、抖动、环境和框架问题。

遇到真实机械臂动作异常时，不要为了“再试一次”连续发送命令。先停止或断电，再回到最小链路检查：上电、USB、串口可见、状态可读、复位可用。
