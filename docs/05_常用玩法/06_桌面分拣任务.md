# 桌面分拣任务

> 状态：后续支持。本页先保留任务设计入口和资料规划，当前还不是完整可直接跑通的分拣教程。

桌面分拣任务适合课程实验、比赛项目和综合展示。它把机械臂动作、摄像头、物块、桌面场景和任务规则组合起来，比单独运行脚本更接近真实项目。

这个章节里的“任务”不是单条命令，而是一套可重复执行的流程：设备摆放、目标物、规则、评分和安全边界都要固定下来。

## 目标效果

后续补全本玩法后，你应该能组织一个完整任务：

- 在桌面上摆放若干物块。
- 让机械臂识别、抓取或移动目标物。
- 按颜色、形状、位置或类别完成分拣。
- 用明确规则评价任务完成情况。

## 推荐任务版本

> 后续支持：这里会补充至少一个标准桌面任务版本，并提供设备摆放、物块规格、运行步骤和验收标准。

| 版本 | 设备 | 任务目标 | 难度 |
| --- | --- | --- | --- |
| A. 手动遥操分拣 | 单臂、手柄或 Joy-Con | 通过人工遥操把物块放到目标区域 | 入门 |
| B. 录制动作分拣 | 单臂 | 录制固定动作，重复移动同一位置物块 | 入门 |
| C. 摄像头辅助抓取 | 单臂、摄像头 | 根据画面定位目标物并抓取 | 进阶 |
| D. 主从遥操分拣 | 双臂 | 用 master 控制 slave 完成分拣 | 进阶 |
| E. 数据采集分拣 | 单臂或双臂、摄像头 | 为 LeRobot 或模仿学习采集演示数据 | 科研预备 |

## 场地设计

建议把桌面划分为几个固定区域：

```text
+-------------------------------+
| 摄像头视野 / 任务工作区        |
|                               |
|  [待分拣区]     [目标区 A]     |
|                               |
|                 [目标区 B]     |
+-------------------------------+
```

每次比赛或课程实验前，建议固定：

- 机械臂底座位置。
- 摄像头位置和朝向。
- 待分拣区。
- 目标区。
- 物块尺寸、颜色和数量。
- 断电位置和安全观察线。

## 推荐流程

> 后续支持：这里会补充从桌面布置、摄像头确认、动作路径测试到分拣演示的完整流程。

1. 先完成 [动作录制与复现](02_动作录制与复现.md)，确认机械臂可以稳定重复动作。
2. 再完成 [摄像头抓取案例](05_摄像头抓取案例.md)，确认相机画面和抓取区域。
3. 设计物块和目标区。
4. 写清楚任务规则和评分方式。
5. 空载测试移动路径。
6. 加入物块，低风险运行。
7. 记录成功率、失败原因和改进点。

## 评分样例

| 指标 | 说明 | 示例分值 |
| --- | --- | --- |
| 完成数量 | 成功放入目标区的物块数量 | 40 |
| 稳定性 | 多次运行是否可重复 | 20 |
| 安全性 | 是否碰撞、拉线或越界 | 20 |
| 用时 | 完成任务所需时间 | 10 |
| 工程记录 | 是否记录端口、相机、参数和失败原因 | 10 |

## 常见失败原因

- 相机位置变化，导致识别结果不稳定。
- 物块太重、太滑或尺寸不适合夹爪。
- 目标区太靠近机械臂限位。
- 线材进入底座或关节旋转范围。
- 桌面反光、光照不足或背景干扰。
- 没有固定机械臂底座，重复运行时基准漂移。
