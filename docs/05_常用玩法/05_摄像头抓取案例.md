# 摄像头抓取案例

> 状态：后续支持。本页先保留玩法入口和资料规划，当前还不是完整可直接跑通的抓取教程。

摄像头抓取案例用于把机械臂控制和视觉输入结合起来。它适合课程展示、创客项目和比赛任务，也可以作为后续 LeRobot 数据采集的前置练习。

视觉输入可以先理解成“程序从摄像头画面里获得目标物位置或场景信息”。摄像头编号、摆放位置和光照会直接影响抓取效果。

系统编号是电脑给摄像头分配的名字或序号，例如 Windows 相机软件里的设备名、Linux 的 `/dev/video0`，或者 OpenCV 中的 camera index。它不一定和摄像头摆放位置天然对应，需要人工确认。

## 目标效果

后续补全本玩法后，你应该能：

- 连接并确认摄像头编号。
- 运行仓库中的抓取示例或参考程序。
- 观察相机画面、目标物和机械臂动作之间的关系。
- 记录一个可复现的抓取场景。

## 需要准备

- 一台已经完成 [快速上手](../02_快速上手/README.md) 的机械臂。
- 摄像头，例如腕部摄像头或外置摄像头。
- 待抓取物块。
- 稳定桌面、固定相机和充足光照。

连接前请确认：

- 摄像头可以被系统打开。
- 视野能覆盖任务区域或夹爪工作区域。
- 相机线不会影响机械臂运动。
- 腕部摄像头固定可靠，不会在运动中松动。
- 多摄像头场景下，相机编号和物理位置一一对应。

## 推荐流程

> 后续支持：这里会补充从摄像头安装、画面确认、示例编译、参数配置到第一次抓取的完整步骤。

1. 先确认机械臂可以 `[CMD][6]` 读取状态、`[CMD][1]` 复位。
2. 确认摄像头能打开，并记录系统编号。
3. 固定相机，保证画面能覆盖任务区域。
4. 放置一个低风险、轻量、不会卡住夹爪的物块。
5. 按示例 README 构建并运行对应程序。
6. 观察相机画面、识别结果和机械臂动作。
7. 每次调整相机位置或物块位置后，重新记录场景配置。

Linux 下可以用 `ffplay` 快速确认摄像头画面：

```bash
ls /dev/video*
ffplay -f v4l2 /dev/video0
```

`/dev/video0` 是 Linux 下的摄像头设备名，`ffplay` 只负责预览画面。多摄像头时需要逐个打开，确认编号和物理位置对应。

## 成功现象

- 摄像头画面能覆盖目标物和夹爪工作区域。
- 示例程序能打开摄像头。
- 机械臂能移动到预期区域附近。
- 抓取过程没有碰撞桌面、相机支架或线材。
