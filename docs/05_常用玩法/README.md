# 常用玩法与项目案例

本章把前面章节里的底层能力组织成可演示、可上课、可比赛的小项目。它不重复讲完整固件指令、SDK、ROS 2 或 LeRobot 细节，而是告诉你“现在能做什么、需要准备什么、从哪里继续”。

如果你还没有完成真实机械臂的第一次动作，请先阅读 [快速上手](../02_快速上手/README.md)。如果你还不熟悉串口指令，请先阅读 [基础操作](../04_基础操作/README.md)。

## 先认识这些玩法组件

| 组件 | 本章里的作用 |
| --- | --- |
| Web 仿真 | 在浏览器中观察模型和交互，不一定驱动真机 |
| tools 脚本 | 仓库提供的日常工具，用于遥操、状态读取或链路验证 |
| examples 示例 | 面向抓取、视觉或桌面任务的参考程序 |
| 外部输入设备 | 手柄、Joy-Con 或主臂等用于实时控制的输入来源 |
| 摄像头 | 给抓取、分拣和数据采集提供视觉输入 |

如果不熟悉这些词，可以先看 [术语与工具速查](../术语与工具速查.md)。

## 推荐路线

第一次做项目案例，建议从低风险、容易观察结果的路线开始：

```text
Web 仿真体验
  -> 动作录制与复现
  -> 手柄遥操
  -> 主从臂遥操
  -> 摄像头抓取案例
  -> 桌面分拣任务
```

课程或比赛可以按“展示效果”组织：

```text
基础动作演示
  -> 可重复动作表演
  -> 人工遥操完成任务
  -> 摄像头辅助抓取
  -> 桌面分拣挑战
```

## 使用原则

- 每个玩法都先空载、低风险验证，再加入物块、夹取和任务道具。
- 涉及真实机械臂运动时，始终保留可以快速断电的方式。
- 涉及双臂时，先完成 [主从臂遥操](04_主从臂遥操.md) 的设备记录；涉及摄像头时，先完成 [摄像头抓取案例](05_摄像头抓取案例.md) 的相机编号确认。
- 涉及脚本或示例程序时，先确认单臂可以通过 `[CMD][6]` 读到状态、通过 `[CMD][1]` 复位。
- 遇到动作异常、相机编号不一致或串口不稳定，优先回到 [校准维护与排障](../08_校准维护与排障/README.md)。
