# Web 仿真体验

Web 仿真适合作为第一个可视化玩法：不用先驱动真实机械臂，就能在浏览器里观察模型、关节和拖拽交互。它可以帮助学习者先理解机械臂结构，再进入真机控制。

这里的 Web 仿真指浏览器里的模型显示和交互，不等同于已经接入真实机械臂。它适合先确认模型、资源和前端环境。

## 目标效果

完成本玩法后，你应该能：

- 打开 Web 控制端或仿真页面。
- 在页面中看到机械臂模型。
- 观察关节、连杆和夹爪的相对位置。
- 通过页面交互理解后续真机动作和姿态变化。

## 需要准备

- 一台可以运行 Web 项目的电脑。
- Node.js、pnpm 等前端环境。
- 当前仓库代码。

具体环境准备见 [安装与准备](../03_安装与准备/README.md)，Web 项目说明见 [Web 接入](../06_仿真与框架接入/Web/README.md) 和 [web/README.md](../../web/README.md)。

## 推荐流程

进入 Web 项目：

```bash
cd web
pnpm install
pnpm run dev
```

`pnpm install` 用于安装 Web 依赖，`pnpm run dev` 用于启动本地开发服务器。终端输出的 `localhost` 地址就是浏览器要打开的页面。

启动后，根据终端输出打开本地页面。第一次体验建议按下面顺序观察：

1. 确认页面能正常加载。
2. 确认机械臂模型完整显示，没有缺失连杆或夹爪。
3. 观察底座、6 个关节和夹爪的大致位置。
4. 尝试页面中已有的拖拽或控制交互。
5. 对照真实机械臂，理解“模型姿态”和“真机姿态”的关系。

## 成功现象

- 浏览器中可以看到完整机械臂模型。
- 页面交互时模型有响应。
- 浏览器控制台没有持续刷出资源缺失或模型加载失败错误。

## 常见问题

| 现象 | 优先检查 |
| --- | --- |
| 页面打不开 | 前端依赖是否安装完成、开发服务器是否启动 |
| 模型不显示 | URDF 或模型资源路径是否正确 |
| 页面很卡 | 浏览器是否支持 WebGL、电脑显卡驱动是否正常 |
| 不知道下一步怎么接真机 | 继续阅读 [仿真与框架接入](../06_仿真与框架接入/README.md) |
