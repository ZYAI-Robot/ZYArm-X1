# 功能总览

本页先用“用户能做什么”建立能力地图，再列出底层固件命令作为依据。第一次了解项目时，可以先看能力地图；需要做协议或 SDK 开发时，再看固件命令表。

> 注意：命令 ID 以 [firmware/Core/Inc/arm_shell.h](../../firmware/Core/Inc/arm_shell.h) 中的 `CMD_ID_*` 枚举为准。当前存在保留/未注册 ID，不建议按连续编号猜测命令。

## 用户能力地图

```text
基础控制
  -> 状态读取 / 关节控制 / 夹爪控制 / 速度设置 / 逆解

玩法体验
  -> Web 仿真 / 主从遥操 / 手柄遥操 / 动作录制与回放 / 视觉抓取

框架接入
  -> Serial API / Python SDK / C++ SDK / ROS 2 / MoveIt / Gazebo / LeRobot

科研实验
  -> 100Hz 级控制链路 / 遥操采集 / 数据集回放 / 策略评估 / VLA 接入
```

## 固件命令能力

底层固件命令来自 [firmware/Core/Src/arm_shell_cmd.c](../../firmware/Core/Src/arm_shell_cmd.c) 的 `g_shell_cmd_list`。

### 运动与姿态控制

| 命令 | ID | 能力 |
| --- | ---: | --- |
| `ik_inverse` | 0 | 根据 `x y z rx ry rz` 做运动学逆解，可选 verbose 输出 |
| `reset` | 1 | 回到初始姿态 |
| `stop` | 2 | 指令急停，立即停止当前运动并锁定关节 |
| `joint_sync` | 3 | 同步设置六个关节和夹爪，支持 `sync verbose fast` 参数 |
| `set_joint` | 5 | 设置单个关节角度 |
| `zero` | 7 | 将当前位置设为零点 |
| `set_claw` | 8 | 设置夹爪位置 |
| `remote_mode` | 24 | 开启或关闭遥控模式 |
| `remote_reset` | 25 | 遥控模式下复位到初始姿态 |
| `remote` | 26 | 按 `x y z rx ry rz claw_angle` 遥控末端位姿 |
| `joint_io_fast` | 36 | 批量读取关节快照、快速同步下发目标并输出 `[STATUS]` |

### 状态读取与参数查询

| 命令 | ID | 能力 |
| --- | ---: | --- |
| `version` | 4 | 读取固件版本信息 |
| `status` | 6 | 查询所有关节和夹爪当前状态，可选 verbose |
| `get_claw` | 9 | 读取当前夹爪位置 |
| `get_joint` | 10 | 读取指定关节角度 |
| `get_speed` | 12 | 读取当前关节速度 |
| `status_report` | 17 | 开启或关闭周期状态上报，可设置频率 |
| `get_status_report` | 18 | 读取周期状态上报状态和频率 |
| `get_name` | 22 | 读取机械臂名称 |
| `get_weights` | 27 | 读取 IK 关节权重 |
| `get_angle_threshold` | 30 | 读取遥控角度阈值 |
| `get_transport_stats` | 35 | 读取 UART1/UART6 通信健康快照 |

### 速度、权重与维护

| 命令 | ID | 能力 |
| --- | ---: | --- |
| `set_speed` | 11 | 设置关节速度，单位为 `degree/s` |
| `set_name` | 21 | 设置机械臂名称 |
| `poweroff` | 23 | 关闭机械臂电源 |
| `set_weights` | 28 | 设置 IK 关节权重 |
| `set_angle_threshold` | 29 | 设置遥控角度阈值 |

### 动作录制与回放

| 命令 | ID | 能力 |
| --- | ---: | --- |
| `record` | 13 | 按名称录制当前运动 |
| `record_player` | 14 | 按名称回放已录制运动 |
| `record_stop` | 15 | 停止录制 |
| `record_list` | 16 | 列出已录制动作 |
| `record_clear` | 19 | 删除指定录制动作 |
| `record_show` | 20 | 查看指定录制动作 |

### 主从遥操

| 命令 | ID | 能力 |
| --- | ---: | --- |
| `master_slave` | 32 | 启动主从跟随模式，支持 master/slave 角色和频率参数 |
| `master_slave_stop` | 33 | 停止主从跟随模式 |
| `master_slave_set_lpf` | 34 | 设置主从跟随低通滤波系数 |

## 上层软件能力

这些能力不是单个固件命令，而是由固件命令、SDK、Web、ROS 2、Gazebo 或 LeRobot 组合出来的用户工作流。

### 基础控制

- 状态读取：`status`、`status_report`、SDK `query_state()` / `get_latest_state()`
- 关节控制：`set_joint`、`joint_sync`、SDK `fast_io()`
- 夹爪控制：`set_claw`、`get_claw`
- 速度设置：`set_speed`、`get_speed`
- 运动学逆解：`ik_inverse`、SDK/ROS 运动学接口
- 动作录制：`record`、`record_stop`、`record_list`
- 动作回放：`record_player`、`record_show`、`record_clear`

### 遥操与玩法

- 主从臂遥操：`master_slave`、`master_slave_stop`、`master_slave_set_lpf`
- 手柄遥操：`remote_mode`、`remote_reset`、`remote`
- Web 拖动与仿真：Web 项目和 URDF/Three.js 链路
- 摄像头抓取案例：`software/examples/` 中的视觉抓取示例
- 桌面分拣任务：由基础控制、视觉和动作录制组合实现

### 框架接入

- Serial API
- Python SDK
- C++ SDK
- Web
- ROS 2
- MoveIt
- Gazebo
- LeRobot

### 科研与数据采集

- 100Hz 级状态读取和控制链路：`joint_io_fast`、SDK `fast_io()`、LeRobot 插件路径
- LeRobot 遥操：`lerobot_robot_zyarm` + `zyarm_sdk`
- 数据集录制：LeRobot `record` 流程
- 数据集回放：LeRobot `replay` 流程
- 策略评估：LeRobot policy eval 路径
- 摄像头配置：LeRobot camera 配置和 Web/示例程序
- 强化学习、模仿学习和 VLA 实验接口：SDK、LeRobot 和上层算法框架
