# 软件生态

ZYArmV1 的软件生态从底层固件到上层算法框架分层组织。

```text
真实机械臂
  <- firmware
  <- Serial API
  <- Python/C++ SDK
  <- Web / ROS 2 / MoveIt / Gazebo / LeRobot
  <- 应用、课程、比赛和科研算法
```

## 三条主路线

```text
低门槛控制路线
  Python SDK / Serial API -> firmware -> 真实机械臂

仿真与工程路线
  ROS 2 / MoveIt / Gazebo / Web -> 模型、规划、仿真和可视化

科研数据路线
  LeRobot -> zyarm_sdk -> 串口 -> 数据采集、回放和策略评估
```

## 先看懂层级

| 层级 | 它解决什么问题 | 适合什么时候看 |
| --- | --- | --- |
| 固件 / Serial API | 最底层通信和动作执行 | 想确认真机链路、理解 `[CMD]` 指令 |
| SDK | 用程序调用接口，不直接拼串口文本 | 想写 Python/C++ 脚本或工具 |
| Web | 浏览器可视化和网页交互 | 想看模型、做演示或开发控制端 |
| ROS 2 / MoveIt / Gazebo | 机器人系统、规划和仿真 | 想做工程化控制、规划或仿真 |
| LeRobot | 遥操、数据采集和策略评估 | 想做科研数据和学习算法 |

如果某个工具不熟悉，可以先查 [术语与工具速查](../术语与工具速查.md)。

## 固件

`firmware/` 是 STM32 固件工程，负责串口命令解析、关节控制、状态输出和底层执行。

适合关注：

- 串口命令实现
- 关节控制执行
- 状态输出
- 固件协议扩展

## SDK

`software/zyarm_sdk/` 提供 Python/C++ SDK。它不依赖 ROS 2，适合教学脚本、诊断工具、LeRobot 插件和二次开发。

适合关注：

- 快速写脚本控制真机
- 高频 `fast_io()` 控制
- 状态缓存与查询
- Python/C++ 二次开发

## Web

`web/` 是 Web 控制端工程，使用 Vue、Three.js、URDF Loader 等技术栈，适合可视化、拖动和网页仿真方向。

适合关注：

- 浏览器中的机械臂模型展示
- Web 拖动和交互
- 课堂演示和低门槛可视化

## ROS 2 / MoveIt / Gazebo

`software/ros2_ws/` 承载模型显示、ros2_control、MoveIt 真机控制、Gazebo 仿真和 ROS 侧硬件访问。

适合关注：

- ROS 2 包开发
- MoveIt 规划
- ros2_control 真机控制
- Gazebo 仿真和 RViz 可视化

## LeRobot

`software/lerobot_robot_zyarm/` 是 LeRobot 原生适配插件，不依赖 ROS 2。它通过 `zyarm_sdk` 直接访问串口，服务主从遥操、数据采集、回放和策略评估。

适合关注：

- 主从臂遥操
- 摄像头数据采集
- LeRobot dataset
- 策略回放和评估

## 示例和诊断

- `software/examples/`：抓取、摄像头和参考场景
- `software/tools/`：日常调试、主从遥操、手柄控制等工具
- `software/diagnostics/`：压测、可靠性验证和诊断脚本
