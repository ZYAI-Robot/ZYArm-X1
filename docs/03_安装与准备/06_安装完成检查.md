# 安装完成检查

本页用于在进入具体玩法或框架前做最后确认。不同目标只需要检查对应部分。

如果某个检查项里的工具名不熟悉，先回到对应环境页面，或查 [术语与工具速查](../术语与工具速查.md)。本页只做最终确认，不替代安装教程。

## 只跑通真机串口

- [ ] 机械臂已经正确上电。
- [ ] 已知道如何快速切断机械臂电源。
- [ ] Windows 能看到 `COMx`，或 Ubuntu 能看到 `/dev/ttyUSBx` / `/dev/ttyACMx`。
- [ ] 串口参数为 `230400, 8N1, no flow control`。
- [ ] 可以发送 `[CMD][6]` 并看到 `[STATUS]`。
- [ ] 可以发送 `[CMD][1]` 让机械臂复位。

下一步：[快速上手](../02_快速上手/README.md) 或 [基础操作](../04_基础操作/README.md)。

## Python SDK

- [ ] Python 版本满足 SDK 要求。
- [ ] 已激活虚拟环境。
- [ ] 已执行 `pip install -e software/zyarm_sdk/python`。
- [ ] `python -c "import zyarm_sdk; print('zyarm_sdk ok')"` 通过。
- [ ] `get_arm_info.py` 可以打开目标串口。

下一步：[仿真与框架接入 / Python](../06_仿真与框架接入/Python/README.md)。

## Web

- [ ] Node.js 和 pnpm 可用。
- [ ] 已在 `web/` 中执行 `pnpm install`。
- [ ] `pnpm run dev` 可以启动开发服务器。
- [ ] 浏览器可以打开页面。
- [ ] 机械臂模型可以正常显示。

下一步：[Web 仿真体验](../05_常用玩法/01_Web仿真体验.md) 或 [Web 接入](../06_仿真与框架接入/Web/README.md)。

## ROS 2 / MoveIt / Gazebo

- [ ] Ubuntu 环境可用。
- [ ] `source /opt/ros/jazzy/setup.bash` 后 `ros2` 命令可用。
- [ ] `colcon` 命令可用。
- [ ] `software/ros2_ws` 可以完成 `colcon build --symlink-install`。
- [ ] `source install/setup.bash` 后能看到 zyarm 相关包。
- [ ] 只看模型显示的 launch 可以启动。

下一步：[仿真与框架接入](../06_仿真与框架接入/README.md)。

## LeRobot

- [ ] Python 版本满足 LeRobot 插件要求。
- [ ] 已安装 `lerobot==0.5.1`。
- [ ] 已安装 `software/zyarm_sdk/python`。
- [ ] 已安装 `software/lerobot_robot_zyarm`。
- [ ] `import lerobot_robot_zyarm` 通过。
- [ ] leader/follower 串口、名称和物理标签已经记录。
- [ ] 摄像头编号和物理位置已经确认。

下一步：[科研与数据采集](../07_科研与数据采集/README.md)。

## 固件开发

- [ ] Keil 可以打开 `firmware/MDK-ARM/ArmV1.uvprojx`。
- [ ] STM32F4 设备支持包已安装。
- [ ] 工程可以编译。
- [ ] 烧录或调试器可以识别控制板。
- [ ] 修改固件前已记录当前版本和验证方式。

下一步：[开发者指南](../09_开发者指南/README.md)。

## 遇到问题时

如果复杂框架无法运行，先回退检查：

```text
机械臂上电
  -> 串口可见
  -> CMD6 可读状态
  -> CMD1 可复位
  -> Python 或框架环境验证
  -> 目标玩法或实验
```

排障入口：[校准维护与排障](../08_校准维护与排障/README.md)。
