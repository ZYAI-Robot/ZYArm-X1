# 固件开发环境

固件开发环境用于打开、编译和烧录 STM32 固件工程。本页只说明环境准备边界；固件架构、协议扩展和代码修改说明后续放在 [开发者指南](../09_开发者指南/README.md)。

## 先认识这些工具

| 工具/术语 | 本页用它做什么 |
| --- | --- |
| 固件 | 运行在控制板上的底层程序 |
| STM32 | 控制板使用的微控制器系列 |
| Keil MDK-ARM | 打开和编译 `.uvprojx` 固件工程 |
| Pack / DFP | Keil 中的芯片支持包，提供 STM32F4 设备文件 |
| 调试器 | 用于烧录固件或调试控制板 |

## 适用场景

- 修改固件命令。
- 调整底层控制逻辑。
- 编译 STM32 固件。
- 使用调试器烧录或调试控制板。

如果只是使用机械臂，不需要安装固件开发环境。

## 工程位置

当前固件工程位于：

```text
firmware/
```

Keil 工程文件位于：

```text
firmware/MDK-ARM/ArmV1.uvprojx
```

从工程文件可见，当前目标芯片为：

```text
STM32F407VETx
```

工程使用的 Pack 信息包括：

```text
Keil.STM32F4xx_DFP.3.1.1
```

## 需要准备

| 项目 | 用途 |
| --- | --- |
| Keil MDK-ARM | 打开和编译 `.uvprojx` 工程 |
| STM32F4 设备支持包 | 提供 STM32F407 设备文件 |
| 烧录或调试器 | 将固件写入控制板 |
| USB 驱动 | 让调试器和控制板被 Windows 识别 |

具体 Keil 版本、调试器型号和烧录流程需要项目方确认。

## 最小检查

在 Windows 上完成环境后，建议检查：

- Keil 可以打开 `firmware/MDK-ARM/ArmV1.uvprojx`。
- 工程目标为 `ArmV1`。
- 设备选择为 `STM32F407VETx`。
- STM32F4 相关 Pack 已安装。
- 工程可以完成一次编译。
- 调试器可以识别目标板。

## 编译成功参考

在 Keil 中点击编译后，如果 Build Output 中出现 `0 Error(s)`，说明工程已经完成一次基础编译。

## 烧录工具入口

当前固件烧录建议使用 ATK-XISP 工具。它用于把 Keil 编译生成的固件写入控制板。

下载地址：

```text
http://www.openedv.com/docs/tool/ruanjian/ATK-XISP.html
```

本页只确认烧录工具入口；具体接线方式、BOOT 配置、串口选择、固件文件路径和烧录后版本确认，会放到 [开发者指南](../09_开发者指南/README.md) 的固件开发章节展开。

## 修改前建议

- 先确认当前固件版本和机械臂状态读取正常。
- 修改前保存对应固件版本说明。
- 不要在不了解协议影响的情况下修改已有 `CMD` ID。
- 新增或修改固件指令后，同步更新 [基础操作](../04_基础操作/README.md) 和 Serial API 文档。

## 常见问题

| 现象 | 优先检查 |
| --- | --- |
| Keil 打不开工程 | MDK 版本、路径、工程文件是否完整 |
| 缺少设备包 | STM32F4xx DFP 是否安装 |
| 编译失败 | Include path、Pack、生成脚本、工具链版本 |
| 无法烧录 | 调试器驱动、接线、供电、芯片型号 |
| 烧录后串口无响应 | 固件版本、串口配置、控制板复位、硬件连接 |
