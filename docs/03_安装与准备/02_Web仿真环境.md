# Web 控制台使用环境

本页说明使用官方在线 Web 控制台前需要准备什么、怎么确认浏览器和串口条件，以及官网页面不可用时应该怎样切换备用路线。

官方 Web 控制台地址：

```text
https://arm.zyairobot.com/#/zy
```

Web 控制台由项目方部署在官网服务器上。学习者只需要按本页确认浏览器、串口、供电和安全空间，就可以开始连接真实机械臂。

## 这页解决什么问题

| 你想做什么 | 推荐做法 |
| --- | --- |
| 第一次连接机械臂 | 使用官方 Web 控制台 |
| 课堂或现场演示 | 提前确认现场网络可以访问官网，并准备串口工具备用 |
| 学习关节控制、动作录制、动作编排 | 使用官方 Web 控制台，再进入 [常用玩法](../05_常用玩法/README.md) |
| 官网页面打不开 | 先排查网络；如果短时间无法恢复，切换到串口工具或 SDK 路线 |

Web 页面能打开，不代表机械臂已经连接成功。只要要控制真实机械臂，仍然需要机械臂上电、USB 串口可见、浏览器支持 Web Serial、串口没有被其他程序占用，并且动作前确认安全空间。

## 需要准备

| 项目 | 要求 | 说明 |
| --- | --- | --- |
| 浏览器 | 推荐 Chrome 或 Edge | Web Serial 在这些浏览器中支持最好 |
| 页面地址 | `https://arm.zyairobot.com/#/zy` | Web Serial 需要安全页面，务必使用官方入口 |
| 网络 | 能访问官网服务器 | 课堂或比赛现场建议提前测试 |
| USB 数据线 | 支持数据传输 | 只能充电的线无法识别串口 |
| 串口驱动 | Windows 可能需要 CH340 驱动 | 驱动异常时页面和串口工具都看不到设备 |
| 机械臂电源 | 真机动作前必须上电 | 页面打开不等于机械臂已可控 |
| 安全空间 | 夹爪、连杆和线材周围保持空旷 | Web 中的部分按钮会真实驱动机械臂 |

准备好这些条件后，就可以直接打开官网页面。

## 打开并连接

1. 使用 Chrome 或 Edge 打开官方地址：

```text
https://arm.zyairobot.com/#/zy
```

2. 给机械臂上电，并用 USB 数据线连接电脑。
3. 在系统里确认能看到串口，例如 Windows 下的 `COM5`。
4. 确认没有串口助手、Keil、Python 脚本或其他浏览器页面占用同一串口。
5. 回到 Web 页面，点击右上角“请选择设备”。
6. 在浏览器弹出的串口选择窗口中选择机械臂对应串口。
7. 右上角显示“设备已连接”后，再进行真机操作。

<img src="../assets/Images/web_连接设备成功.png" alt="Web 控制台设备连接成功" width="960">

连接失败时，优先检查浏览器、HTTPS 页面、Web Serial 权限、CH340 驱动、USB 数据线、机械臂上电和串口占用。

## 验证清单

- 能打开官方 Web 地址。
- 正在使用 Chrome 或 Edge。
- 机械臂已经上电。
- 电脑能看到机械臂串口。
- 串口没有被其他程序占用。
- 页面可以弹出串口选择窗口。
- 右上角可以显示“设备已连接”。
- 页面状态没有持续异常。
- 已知道如何点击“停止”和如何直接切断机械臂电源。

## 官网不可用时怎么办

如果官方 Web 控制台暂时打不开，建议按下面顺序处理：

1. 换网络、关闭代理或更换浏览器后重试官方地址。
2. 如果现场必须继续教学或排障，使用串口工具完成最小闭环：`CMD6` 读状态、`CMD1` 复位、低风险小动作、再次复位。
3. 如果你已经准备好 Python 环境，可以使用 Python SDK 或 tools 做状态读取和复位验证。
4. 记录失败现象、浏览器版本、网络环境和页面提示，反馈给项目方。

串口备用路线见 [快速上手](../02_快速上手/README.md) 和 [基础操作](../04_基础操作/README.md)。Python 路线见 [Python SDK 环境](01_Python_SDK环境.md)。

## 常见问题

| 现象 | 优先检查 |
| --- | --- |
| 官方 Web 打不开 | 网络、DNS、浏览器代理、现场网络策略 |
| 页面打开后找不到串口 | 是否使用 Chrome/Edge、USB 线是否支持数据、驱动是否正常 |
| 串口选择窗口里没有机械臂 | 机械臂是否上电、设备管理器是否能看到 `COMx`、串口是否被占用 |
| 连接后状态异常 | 供电、USB 线、固件运行状态、右侧控制台提示 |
| 浏览器提示权限失败 | 是否拒绝了串口授权，是否在官方 HTTPS 页面中打开 |
| 页面模型显示异常 | 浏览器 WebGL 支持、显卡驱动、页面资源是否加载完整 |
