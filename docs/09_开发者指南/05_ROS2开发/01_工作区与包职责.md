# 工作区与包职责

ROS 2 工作区源码位于 `software/ros2_ws/src/`。`build/`、`install/`、`log/` 是构建产物，维护代码时优先关注 `src/`。

## 包职责

| 包 | 职责 |
| --- | --- |
| `zyarm_description` | 机械臂模型、网格、URDF/Xacro、RViz 配置 |
| `zyarm_control` | `ros2_control` 控制器配置 |
| `zyarm_interfaces` | 共享 srv/msg/action 定义 |
| `zyarm_hardware` | Python 硬件运行时、多机械臂实例、串口服务、纯 ROS 遥操 |
| `zyarm_hardware_interface` | C++ ros2_control 真机硬件插件 |
| `zyarm_moveit_config` | MoveIt SRDF、运动学、规划器、RViz 配置 |
| `zyarm_bringup` | 系统级启动入口 |
| `zyarm_gazebo` | Gazebo 场景、模型、launch 和仿真辅助脚本 |

## 修改定位

| 需求 | 首要包 |
| --- | --- |
| 修改模型、mesh、joint 名称 | `zyarm_description` |
| 修改控制器配置 | `zyarm_control` |
| 新增 ROS service | `zyarm_interfaces` |
| 修改服务式真机控制或纯 ROS 主从 | `zyarm_hardware` |
| 修改 MoveIt 真机控制链 | `zyarm_hardware_interface`、`zyarm_moveit_config` |
| 修改系统启动入口 | `zyarm_bringup` |
| 修改仿真场景 | `zyarm_gazebo` |

## 构建和测试建议

- 改模型后，先跑模型显示 launch。
- 改接口后，重新构建依赖该接口的包。
- 改 launch 后，增加或更新 launch 测试。
- 改真机控制前，先在假硬件或仿真链路验证。
