# 插件结构

LeRobot 插件源码位于 `software/lerobot_robot_zyarm/`。它是 Python 包，通过 LeRobot 的 robot/teleoperator 机制接入 ZYArm。

如果只是使用插件采集数据，不需要先理解本页所有源码。使用路径见 [科研与数据采集](../../07_科研与数据采集/README.md)。

## 关键文件

| 文件 | 作用 |
| --- | --- |
| `pyproject.toml` | Python 包配置 |
| `src/lerobot_robot_zyarm/config.py` | robot/teleoperator 配置 |
| `src/lerobot_robot_zyarm/features.py` | observation/action 特征定义 |
| `src/lerobot_robot_zyarm/conversion.py` | SDK 与 LeRobot 数据表达转换 |
| `src/lerobot_robot_zyarm/robot.py` | follower 机械臂适配 |
| `src/lerobot_robot_zyarm/teleoperator.py` | leader 主臂适配 |
| `tests/` | 插件单元测试 |

## 包导出

`__init__.py` 对外导出配置类和适配类：

```python
from .robot import ZyArmFollowerRobot
from .teleoperator import ZyArmLeaderTeleoperator
```

这样 LeRobot 在解析 `--robot.type=zyarm_follower` 或 `--teleop.type=zyarm_leader` 时，可以通过注册机制找到对应配置和实现。

## 依赖关系

插件依赖 `zyarm_sdk`，不直接依赖 ROS 2、MoveIt 或 Gazebo。如果需要改变控制语义，优先检查 SDK 是否应该提供公共能力。

```text
LeRobot CLI
  -> config.py 注册类型
  -> robot.py / teleoperator.py
  -> conversion.py
  -> zyarm_sdk
  -> 串口 / 固件
```

## 源码职责

| 源码 | 开发时重点关注 |
| --- | --- |
| `config.py` | 类型注册、串口参数、相机配置、频率和滤波参数 |
| `features.py` | LeRobot 看到的 action / observation 字段 |
| `conversion.py` | LeRobot 字典和 SDK public positions 的互转 |
| `robot.py` | follower 连接、读状态、读相机、下发 action |
| `teleoperator.py` | leader 连接、读取输入、retarget、生成 action |
| `tests/` | 修改后最小验证 |

## 特征表达

```text
joint0.pos
joint1.pos
joint2.pos
joint3.pos
joint4.pos
joint5.pos
joint6.pos
```

`joint0.pos` 到 `joint5.pos` 使用 SDK/ROS 公共角度表达，单位为弧度；`joint6.pos` 是夹爪归一化位置，范围为 `0.0..1.0`。

字段细节见 [features 与 conversion 数据字段](03_features与conversion数据字段.md)。
