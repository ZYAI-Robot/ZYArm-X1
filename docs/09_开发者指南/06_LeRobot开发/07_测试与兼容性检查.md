# 测试与兼容性检查

修改 LeRobot 插件后，不能只看代码能否导入。至少要确认 config、字段转换、follower、leader 和最小真实链路没有被破坏。

## 测试目录

测试位于：

```text
software/lerobot_robot_zyarm/tests/
```

| 测试 | 覆盖内容 |
| --- | --- |
| `test_config.py` | `zyarm_follower` / `zyarm_leader` 注册名、默认值 |
| `test_conversion.py` | `ACTION_KEYS`、字段顺序、夹爪归一化、相机 feature |
| `test_robot.py` | follower connect、observation、slave filter、`fast_io()`、disconnect |
| `test_teleoperator.py` | leader connect、`get_action()`、retarget、feedback、disconnect |

## 推荐测试命令

在仓库根目录执行：

```bash
pytest software/lerobot_robot_zyarm/tests
```

如果环境中没有安装测试依赖，先按项目当前 Python 环境安装策略处理。不要为了跑测试在文档里新增运行时依赖。

## 按修改类型选择测试

| 修改类型 | 至少检查 |
| --- | --- |
| 改 `config.py` | `test_config.py`，并确认 README 参数说明同步 |
| 改 `features.py` | `test_conversion.py`，并确认 dataset 字段兼容性 |
| 改 `conversion.py` | `test_conversion.py`、`test_robot.py`、`test_teleoperator.py` |
| 改 `robot.py` | `test_robot.py`，并做短时间 teleoperate / record 验证 |
| 改 `teleoperator.py` | `test_teleoperator.py`，并做短时间 teleoperate 验证 |
| 改相机字段 | `test_conversion.py`、`test_robot.py`，并检查 [摄像头与视角配置](../../07_科研与数据采集/04_摄像头与视角配置.md) |
| 升级 LeRobot | 全部测试，外加 teleoperate、record、replay 最小链路 |

## 最小真实链路验证

下面只是源码修改后的验证思路，不替代完整使用教程。完整流程见 [科研与数据采集](../../07_科研与数据采集/README.md)。

1. 插件能导入。
2. `lerobot-teleoperate` 能打开 leader/follower。
3. follower 能小幅跟随，且可以安全停止。
4. `lerobot-record` 能生成短 episode。
5. `lerobot-replay` 能回放同一 episode。
6. 如果改了 policy 接口，确认 `policy.path` 评估路径能启动。

## 兼容性检查

改动会影响数据集或策略时，额外检查：

- 旧 dataset 是否还能 replay。
- 新 dataset 是否字段完整。
- policy 输出维度是否和 action 字段一致。
- 相机字段名是否和训练脚本一致。
- `software/lerobot_robot_zyarm/README.md` 是否需要同步。
- [科研与数据采集](../../07_科研与数据采集/README.md) 是否需要同步。
- [测试发布与贡献检查](../08_测试发布与贡献检查.md) 是否需要补真实设备验证记录。
