# 固件工程结构

固件目录主要由 STM32Cube 生成代码、机械臂业务代码、Keil 工程和第三方/中间件组成。开发时优先阅读业务代码，不要随意整理 Keil 工程内部结构。

## 关键目录

| 目录 | 作用 |
| --- | --- |
| `firmware/Core/Inc/` | 头文件、模块接口、配置结构 |
| `firmware/Core/Src/` | 业务逻辑、外设初始化、FreeRTOS 任务 |
| `firmware/MDK-ARM/` | Keil 工程、编译配置、输出目录 |
| `firmware/Drivers/` | STM32 HAL/CMSIS 驱动 |
| `firmware/Middlewares/` | FreeRTOS 等中间件 |

`Drivers/` 和 `Middlewares/` 通常不作为业务开发首选修改点。除非升级芯片库或 RTOS，否则应优先在 `Core/Inc` 和 `Core/Src` 内定位问题。

## 串口命令相关文件

| 文件 | 作用 |
| --- | --- |
| `firmware/Core/Inc/arm_shell.h` | `CMD_ID` 枚举、命令结构、解析格式和 shell API |
| `firmware/Core/Src/arm_shell.c` | 原始串口数据解析、命令队列、执行器、ACK/响应输出 |
| `firmware/Core/Src/arm_shell_cmd.c` | 命令表、每个 `[CMD]` 的 handler、help 输出 |
| `firmware/Core/Src/arm_shell_transport.c` | 串口发送通道和优先级相关逻辑 |

`arm_shell_cmd.c` 是理解固件命令的第一入口。文档中的命令编号和名称应以其中的 `g_shell_cmd_list` 为准。

## 机械臂业务模块

| 文件 | 关注点 |
| --- | --- |
| `arm_robot.c` / `arm_robot.h` | 机械臂配置、状态、基础动作 |
| `arm_robot_kinematics.c` / `arm_robot_kinematics.h` | 运动学、逆解和位姿相关逻辑 |
| `arm_robot_recorder.c` / `arm_recorder.h` | 动作录制、回放、列表、删除、展示 |
| `arm_remote.c` / `arm_remote.h` | 遥控模式、遥控复位和远程 IK |
| `arm_master_slave.c` / `arm_master_slave.h` | 主从跟随、频率和滤波 |
| `arm_status_report.c` / `arm_status_report.h` | 自动状态上报 |
| `arm_flash.c` / `arm_flash.h` | 名称、录制动作等 Flash 存储相关逻辑 |

## Keil 工程相关文件

| 文件或目录 | 说明 |
| --- | --- |
| `firmware/MDK-ARM/ArmV1.uvprojx` | 工程配置，新增源码文件时通常需要进入工程配置 |
| `firmware/MDK-ARM/ArmV1.uvoptx` | 用户/工程选项，容易随本地 Keil 环境变化 |
| `firmware/MDK-ARM/ArmV1/` | 编译输出目录，常见产物包括 `ArmV1.hex`、`ArmV1.axf`、`ArmV1.map` |
| `firmware/MDK-ARM/gen_commit_id.bat` | 生成 commit id 相关脚本 |

`ArmV1.uvoptx` 和 `.uvguix.*` 这类文件可能包含本地 IDE 状态。提交前需要确认它们是否属于本次开发目标。

## 开发建议

- 修改已有命令时，先从 `arm_shell_cmd.c` 中对应 handler 读起。
- 新增模块时，同时新增 `.c`、`.h`，并确认 Keil 工程已经包含新源码。
- 不要把上位机协议封装写进固件；固件只负责底层命令和真实行为。
- 每次改变命令参数、回包或单位，都要同步更新 [基础操作](../../04_基础操作/README.md) 和上层开发文档。
