# 固件开发

本节面向需要修改 STM32 固件的维护者。固件是机械臂真实运动的源头，任何命令语义、单位、状态字段或安全行为变化，都可能影响 SDK、Web、ROS 2、LeRobot、示例和诊断脚本。

## 先认识这些工具

| 工具或文件 | 作用 |
| --- | --- |
| Keil MDK | 打开、编译和调试 STM32 工程 |
| ATK-XISP | 常用串口烧录工具，用于把编译产物写入控制板 |
| `.uvprojx` | Keil 工程文件，记录源码、编译选项和目标配置 |
| `.hex` | 常用烧录产物，编译成功后通常用于写入控制板 |
| `[CMD]` | 固件串口命令格式，是上位机控制机械臂的底层入口 |

固件开发环境准备见 [固件开发环境](../../03_安装与准备/05_固件开发环境.md)。本节重点讲工程结构、修改流程和验证检查。

## 固件开发最小闭环

```text
读懂目标 CMD 或模块
  -> 修改固件代码
  -> Keil 编译 0 Error
  -> 烧录到控制板
  -> 串口发送 CMD4/CMD6/help 验证
  -> 运行相关工具或诊断脚本
  -> 同步 docs 和 SDK/框架入口
```

## 安全边界

- 未确认动作语义前，不要把命令描述成安全功能或急停功能。
- 涉及真实运动的命令必须空载、低风险、周围无障碍验证。
- 修改 `CMD0`、`CMD1`、`CMD2`、`CMD3`、`CMD36` 等运动相关命令时，需要同步检查 SDK、工具和诊断脚本。
- 修改状态字段时，需要同步检查 `CMD6`、自动状态上报、SDK mapping、ROS joint state 和 LeRobot observation。
