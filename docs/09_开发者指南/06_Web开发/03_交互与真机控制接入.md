# 交互与真机控制接入

Web 交互可以分为仿真交互和真机控制。仿真交互只改变浏览器中的模型；真机控制会向真实机械臂发送指令，风险等级完全不同。

## 仿真交互

仿真交互常见修改位置：

- `RobotDragControls.js`：拖拽控制。
- `URDFKeyboardController.js`：键盘控制。
- `IKManager.js`：逆解或目标姿态相关逻辑。
- `Menu.vue`：控制按钮、参数输入和模式切换。

修改后要确认 UI 显示、模型动作和状态面板一致。

## 真机控制接入

真机控制应通过明确的通信层进入，不建议在 UI 组件中直接拼接 `[CMD]`。后续可选择：

- WebSocket 到本地服务，再由服务调用 SDK。
- 浏览器串口能力到本地设备。
- 与现有工具或后端服务集成。

无论选择哪条路径，都要明确：

- 仿真模式和真机模式的状态区分。
- 连接、断开、错误、超时和动作完成反馈。
- 真机动作前的安全确认。
- 断电优先的安全说明。
- 与 SDK 单位和固件命令语义的一致性。
