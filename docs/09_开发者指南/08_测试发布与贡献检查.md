# 测试发布与贡献检查

本页用于提交前自查。不是每个改动都需要跑所有项目，但只要涉及协议、单位、状态字段或真实运动，就必须跨层检查。

## 提交前总检查

| 检查项 | 说明 |
| --- | --- |
| 变更范围 | 是否只改了本次任务需要的文件 |
| 模块边界 | 是否把能力放在正确目录 |
| 文档同步 | 用户文档和开发者文档是否一致 |
| 安全说明 | 涉及真实运动时是否写清空载、断电、风险 |
| 待补充占位 | 无法从仓库确认的照片、规格、流程是否保留占位 |
| 测试记录 | 是否记录了编译、脚本、launch 或真机验证结果 |

## 按层检查

| 改动层级 | 最低验证 |
| --- | --- |
| 固件 | Keil 编译、烧录后 `help`、`CMD4`、`CMD6`、目标命令 |
| SDK | Python/C++ 协议测试、mapping 测试、示例或工具运行 |
| ROS 2 | `colcon build --symlink-install`、相关 launch/test |
| LeRobot | 插件导入、短时间 teleoperate/record/replay |
| examples | 依赖安装、构建、最小运行 |
| tools | 参数错误和正常路径都能给出清楚输出 |
| diagnostics | 输出测试条件、失败统计和风险说明 |

## 跨层改动流程

跨层改动指一个功能会同时影响固件、协议、SDK、工具、上层框架或文档。典型例子是新增 `[CMD]`、改变单位、改变状态字段、增加遥操模式或新增数据采集接口。

```text
明确用户场景
  -> 找到首要负责模块
  -> 做最小源码修改
  -> 补调用入口或示例
  -> 补诊断和测试
  -> 同步用户文档
  -> 记录版本和发布说明
```

## 协议和单位变更

如果修改了 `[CMD]`、`[STATUS]`、ACK、单位或状态字段，检查：

- [基础操作](../04_基础操作/README.md)
- [Serial API](../06_仿真与框架接入/Serial_API/README.md)
- [SDK 开发](04_SDK开发/README.md)
- [ROS 2 开发](05_ROS2开发/README.md)
- [LeRobot 开发](06_LeRobot开发/README.md)
- [示例工具与诊断开发](07_示例工具与诊断开发.md)

单位和状态字段属于高风险变更，因为它会影响所有上层调用。修改前先确认：

- 固件输出的是固件角度、用户角度、弧度、角度还是归一化值。
- SDK 是否已有 mapping 层负责转换。
- ROS 2 的 joint state、MoveIt 和 LeRobot feature 是否依赖该字段。
- 诊断脚本是否用该字段判断动作完成或异常。

## 文档同步清单

跨层改动完成后，至少检查：

- [基础操作](../04_基础操作/README.md)：固件命令、参数、回包和注意事项是否变了。
- [仿真与框架接入](../06_仿真与框架接入/README.md)：SDK、Serial API、ROS 2 是否需要更新。
- [科研与数据采集](../07_科研与数据采集/README.md)：LeRobot、数据集、摄像头流程是否受影响。
- [校准维护与排障](../08_校准维护与排障/README.md)：新错误、新日志、新诊断脚本是否需要记录。
- [资料补充清单](../资料补充清单.md)：是否新增需要项目方补充的照片、截图、视频或规格。

## 版本兼容性和回滚

以下情况需要在发布说明中明确提醒：

- 新增、删除或修改 `[CMD]`。
- 改变 `[STATUS]` 字段、顺序、单位或取值范围。
- 改变动作完成 ACK 语义。
- 改变设备名称、录制动作或参数在 Flash 中的存储格式。
- 新固件要求同步升级 SDK、ROS 2 或 LeRobot。

正式烧录或交付新版本前，建议保留上一版稳定固件的 `.hex` 文件、对应 commit/tag 和验证记录。回滚后同样执行版本确认和基础动作验证，不要只看烧录工具显示成功。

## 资料完整性检查

如果文档提到但仓库无法确认下列内容，必须保留 `待项目方补充`：

- 实物外观照片。
- 接口、线材、供电和控制板照片。
- 烧录接线、BOOT 设置和工具截图。
- 相机安装照片、视野样张和推荐型号。
- 官方版本号、发布包、回滚包和下载地址。
- 比赛、课程和科研任务的标准道具、场地和评分规则。

## 待项目方补充

- 正式贡献规范和 PR 模板。
- 发布 checklist 模板。
- 固件、SDK、ROS 2、LeRobot 的版本联动规则。
- `.hex` 发布包命名、保存位置和回滚下载方式。
- 回滚失败或设备无法通信时的恢复流程。
- 测试报告和演示视频的归档位置。
