# 校准维护与排障

本章用于处理使用过程中最常见的问题：找不到串口、读不到状态、机械臂不动作、动作抖动、ROS 2 启动失败、Web 页面异常、LeRobot 采集不顺等。

## 先认识这些排障词

| 术语 | 排障时代表什么 |
| --- | --- |
| 端口 | 电脑识别到的串口或摄像头设备名 |
| 波特率 | 串口通信速度，ZYArm 快速上手使用 `230400` |
| 权限 | Linux 下当前用户是否允许访问串口或摄像头设备 |
| launch | ROS 2 启动文件，用来同时启动多个节点和配置 |
| 依赖 | 程序运行需要的第三方包或系统组件 |
| fps | 摄像头或视频每秒帧数，过高可能导致采集卡顿 |

更多术语见 [术语与工具速查](../术语与工具速查.md)。

## 按现象查找

| 现象 | 优先检查 |
| --- | --- |
| 找不到串口 | USB 线、驱动、设备管理器、`/dev/ttyUSB*`、权限 |
| 能打开串口但没有返回 | 波特率、固件、端口是否选错 |
| 能读状态但不动作 | 安全空间、速度、目标角度、供电 |
| 动作方向不对 | 零位、offset、关节方向、SDK/固件角度表达 |
| 动作卡顿或抖动 | 速度、控制频率、串口稳定性、负载、滤波参数 |
| ROS 2 launch 报错 | ROS 环境、workspace 是否编译、包名和 `x1_standard` 资源 |
| Web 页面无法启动 | Node.js、pnpm、依赖安装、端口占用 |
| LeRobot 无法导入插件 | `lerobot==0.5.1`、editable install、Python 环境 |
| 摄像头打不开 | 摄像头 index、权限、占用、分辨率和 fps |

## 串口连接问题

本节里的“串口号”指 `COMx` 或 `/dev/ttyUSBx` 这类设备名，“权限”主要指 Ubuntu/Linux 当前用户是否允许访问该设备。

1. 确认 USB 线支持数据传输。
2. Windows 查看设备管理器中的 `COMx`。
3. Ubuntu 查看 `/dev/ttyUSB*` 或 `/dev/ttyACM*`。
4. 使用 `230400` 波特率。
5. 先发送 `[CMD][6]` 查询状态。

## 机械臂不动作

“能读状态但不动作”说明上位机到固件的基本通信大概率是通的，问题可能出在动作目标、安全空间、供电、速度或机械结构阻挡上。

- 确认能读到 `[STATUS]`。
- 确认已经设置保守速度。
- 确认目标动作幅度很小。
- 确认机械臂没有接近限位或被外物卡住。
- 如果使用双臂遥操，确认 leader/follower 没有接反。

## 动作卡顿或抖动

卡顿通常和控制频率、串口稳定性、供电或负载有关；抖动还可能和滤波、目标变化过快或机械臂接近限位有关。

- 降低速度和动作幅度。
- 检查供电是否稳定。
- 检查串口线是否松动。
- 高频控制时优先使用 SDK `fast_io()` 或 LeRobot 插件推荐路径。
- 不要在高频循环中混用过多主动状态查询。

## 日志和诊断

诊断脚本位于：

```text
software/diagnostics/
```

日常工具位于：

```text
software/tools/
```

详细说明见：

- [software/diagnostics/README.md](../../software/diagnostics/README.md)
- [software/tools/README.md](../../software/tools/README.md)

`diagnostics` 更偏压测、通信统计和问题定位，`tools` 更偏日常使用、遥操和设备信息读取。遇到复杂问题时，建议先记录 `CMD4`、`CMD6`、`CMD35` 输出，再运行对应脚本。
