# 科研路线与实验准备

科研与数据采集不是从 `record` 命令开始，而是从稳定的设备链路开始。建议先把每一环单独验证，再进入长时间采集或策略评估。

## 推荐路线

```text
跑通真实机械臂
  -> 确认双臂或单臂遥操
  -> 接入摄像头
  -> 运行 LeRobot teleoperate
  -> 运行 LeRobot record
  -> 检查 dataset
  -> replay 数据集
  -> policy eval
```

如果中间某一步不稳定，先停在当前步骤排查，不要继续往后叠加复杂度。

## 需要准备

- 一台 follower 机械臂。它是真正执行动作、被记录 observation 的机械臂。
- 可选的一台 leader 机械臂。它作为人工输入来源，用于主从遥操和演示采集。
- 已确认的串口号，例如 Windows `COM3`、`COM4`，或 Ubuntu `/dev/ttyUSB0`、`/dev/ttyUSB1`。
- 稳定电源和可以快速断电的方式。
- 可选摄像头，例如前置相机、腕部相机或侧视相机。
- Python 环境，以及 LeRobot、`zyarm_sdk`、`lerobot_robot_zyarm`。
- 足够磁盘空间。带视频的数据集会明显占用更多空间。

## 实验前安全检查

- follower 周围没有障碍物，夹爪和连杆运动范围内没有人手或线材。
- leader 和 follower 不会互相碰撞。
- 两台机械臂的电源和 USB 线不会被运动拉扯。
- follower 首次执行 replay 或 policy eval 时保持空载。
- 已确认可以直接切断每台机械臂电源。
- 不在接近限位、碰撞物体或动作异常时继续采集。

## 成功标准

进入正式数据采集前，建议至少达到：

- 每台机械臂都能单独读取状态并复位。
- leader/follower 端口和物理标签一致。
- `lerobot_robot_zyarm` 插件可以被 Python 导入。
- `lerobot-teleoperate` 可以稳定运行一小段时间。
- 相机画面能覆盖任务区域。
- 采集一个短 episode 后可以执行 replay。
