# 常见问题

本页整理科研与数据采集阶段常见问题。处理问题时，优先把系统拆回最小链路：单臂状态读取、双臂端口确认、相机预览、teleoperate，再进入 record 或 replay。

## leader 和 follower 接反

现象：

- 输入端机械臂开始运动。
- follower 不跟随，或动作方向不符合预期。
- replay 时错误机械臂运动。

处理：

- 停止程序，必要时直接断电。
- 用 `CMD22` 或 `get_arm_info.py` 重新确认名称和端口。
- 检查命令里的 `--robot.port` 和 `--teleop.port`。
- 重新填写设备记录表。

## 相机编号错位

现象：

- `front` 显示腕部画面。
- `wrist` 显示桌面远景。
- dataset 里视角和任务描述不匹配。

处理：

- 先用系统相机软件或 `ffplay` 逐个预览。
- 交换 `index_or_path`。
- 重新运行 `lerobot-teleoperate --display_data=true` 确认。

## 控制或画面卡顿

现象：

- 机械臂跟随延迟明显。
- 画面刷新不连续。
- 采集过程中终端持续报编码或相机相关问题。

处理：

- 关闭 `--display_data`。
- 降低相机分辨率和 fps。
- 减少相机数量。
- 调整 `--dataset.encoder_threads`。
- 先做无相机 teleoperate，确认动作链路本身是否稳定。

## Ubuntu 串口权限不足

现象：

- 打开 `/dev/ttyUSB0` 失败。
- 提示 permission denied。

处理：

- 确认设备存在：`ls /dev/ttyUSB*`。
- 将当前用户加入 `dialout` 组后重新登录。
- 临时测试时可用当前系统允许的方式调整设备权限。

## replay 动作和采集时不一致

现象：

- replay 轨迹明显偏离采集动作。
- 回放开始姿态差异大。
- follower 接近限位或碰撞。

处理：

- 停止 replay，保持空载。
- 确认 replay 使用的是正确 follower 端口。
- 先回放短 episode。
- 检查采集时是否端口接反或数据质量差。
- 不要继续使用危险数据。

## policy eval 运动危险

现象：

- 策略输出导致突然大幅动作。
- 夹爪或连杆接近人手、线材或障碍物。
- 动作持续逼近限位。

处理：

- 优先直接断电。
- 缩短 episode。
- 去掉负载和高风险道具。
- 确认训练视角、当前相机视角和任务描述一致。
- 确认 replay 能稳定通过后再评估。

## 不知道该从哪里排查

按下面顺序回退：

```text
CMD6 状态读取
  -> CMD1 复位
  -> 单台机械臂端口确认
  -> 双臂 leader/follower 端口确认
  -> 相机预览
  -> lerobot-teleoperate
  -> 短 episode record
  -> replay
  -> policy eval
```

只要某一步失败，就先停在那一步。
