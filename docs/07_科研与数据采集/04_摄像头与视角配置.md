# 摄像头与视角配置

摄像头用于把任务现场记录进 dataset。常见视角包括前置相机 `front` 和腕部相机 `wrist`。相机编号一旦错位，数据集可能仍然能生成，但后续训练和评估会看到错误视角。

## 先确认物理视角

建议先记录：

| 相机名称 | 系统编号 | 物理位置 | 视野目标 | 备注 |
| --- | --- | --- | --- | --- |
| `front` | `0` 或 `/dev/video0` | 任务区前方 | 桌面、物体、夹爪运动范围 |  |
| `wrist` | `1` 或 `/dev/video1` | 夹爪或腕部 | 夹爪附近细节 |  |

`front` 和 `wrist` 是你给 LeRobot 的逻辑名称，`index_or_path` 是系统或 OpenCV 用来打开相机的编号或路径。两者需要人工对应。

## 预览摄像头

Windows 可以使用系统相机应用、厂商相机软件或其他相机预览工具逐个确认。

Ubuntu 可以先查看视频设备，再逐个预览：

```bash
ls /dev/video*
ffplay -f v4l2 /dev/video0
ffplay -f v4l2 /dev/video1
```

`ffplay` 是一个轻量视频预览工具，适合快速确认 Linux 下哪个 `/dev/video*` 对应哪个物理摄像头。

## LeRobot camera 配置

典型双摄像头配置如下：

```text
--robot.cameras="{front: {type: opencv, index_or_path: 0, width: 640, height: 480, fps: 30}, wrist: {type: opencv, index_or_path: 1, width: 640, height: 480, fps: 30}}"
```

参数含义：

| 参数 | 含义 |
| --- | --- |
| `front` / `wrist` | 数据集里的相机名称 |
| `type: opencv` | 使用 OpenCV 打开相机 |
| `index_or_path` | 相机编号或路径，例如 `0`、`1`、`/dev/video0` |
| `width` / `height` | 图像宽高 |
| `fps` | 目标采集帧率 |

第一次采集建议使用 `640x480, 30fps` 这类较稳妥配置。画面或控制周期卡顿时，优先降低分辨率或 fps。

## 成功现象

- `front` 能看到任务区域和物体。
- `wrist` 能看到夹爪附近或抓取接触区域。
- 相机线不会拉扯机械臂。
- 多摄像头同时打开时，画面没有交换。
- 采集命令启动后，`--display_data=true` 能显示正确画面。

## 常见问题

| 现象 | 可能原因 | 处理方向 |
| --- | --- | --- |
| `front` 和 `wrist` 画面反了 | `index_or_path` 写反 | 交换编号并重新预览 |
| 打开相机失败 | 相机被其他软件占用，或编号不对 | 关闭预览软件，重新确认编号 |
| 采集明显卡顿 | 分辨率/fps 太高，显示或编码压力大 | 降低分辨率、降低 fps、关闭 `display_data` |
| 腕部画面抖动严重 | 相机固定不牢或线材拉扯 | 重新固定相机和线材 |
