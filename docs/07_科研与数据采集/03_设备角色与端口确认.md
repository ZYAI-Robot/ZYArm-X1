# 设备角色与端口确认

LeRobot 数据采集中常见两个角色：

- `leader` / `teleoperator`：提供人工动作输入的主臂。
- `follower` / `robot`：接收动作、执行动作并记录 observation 的从臂。

端口接反会导致采集逻辑混乱，严重时会让不该运动的机械臂运动。运行 LeRobot 命令前，先把物理标签、固件名称和系统端口对齐。

物理标签是贴在设备或线材上的人工标记，固件名称是写入控制板的名称。两者配合使用，可以减少多台设备接反的概率。

## 推荐记录表

| 角色 | 端口 | 固件名称 | 物理标签 | 用途 |
| --- | --- | --- | --- | --- |
| leader | `COM3` 或 `/dev/ttyUSB0` | `master` 或 `leader` |  | 人工输入 |
| follower | `COM4` 或 `/dev/ttyUSB1` | `slave` 或 `follower` |  | 执行动作和采集数据 |

如果同时连接机械臂和摄像头，建议再建立一张完整设备记录表：

| 设备 | 角色 | 系统名称 | 固件名称或相机编号 | 物理标签 | 备注 |
| --- | --- | --- | --- | --- | --- |
| 机械臂 1 | leader / master | `COM3` 或 `/dev/ttyUSB0` | `leader` |  |  |
| 机械臂 2 | follower / slave | `COM4` 或 `/dev/ttyUSB1` | `follower` |  |  |
| 摄像头 1 | front / wrist / side |  | `camera_0` |  |  |
| 摄像头 2 | front / wrist / side |  | `camera_1` |  |  |

## 单台确认

每台机械臂都先单独确认：

```text
[CMD][6]
[CMD][1]
[CMD][22]
```

含义：

| 指令 | 用途 |
| --- | --- |
| `[CMD][6]` | 查询机械臂状态，确认串口能通信 |
| `[CMD][1]` | 复位，确认基础动作正常 |
| `[CMD][22]` | 读取固件名称，用于区分多台机械臂 |

如果需要设置名称，可用：

```text
[CMD][21][leader]
[CMD][22]
```

另一台可设置为：

```text
[CMD][21][follower]
[CMD][22]
```

## 使用工具读取信息

也可以使用工具脚本快速读取名称和版本：

```bash
python software/tools/get_arm_info.py COM3 -b 230400
python software/tools/get_arm_info.py COM4 -b 230400
```

Ubuntu 示例：

```bash
python3 software/tools/get_arm_info.py /dev/ttyUSB0 -b 230400
python3 software/tools/get_arm_info.py /dev/ttyUSB1 -b 230400
```

`-b 230400` 表示串口波特率，需要和固件配置一致。

Ubuntu 长期使用时，可以考虑通过 udev 规则固定设备名，避免重插 USB 后设备编号变化。

`udev` 是 Linux 的设备命名规则机制。它可以把某个 USB 串口或摄像头固定成更稳定的名字，适合长期实验；第一次上手不需要配置。

## 端口接反的典型现象

- 你移动 leader，但 follower 没有跟随。
- 启动命令后，原本作为输入的机械臂开始执行动作。
- replay 时动作发送到了错误的机械臂。
- dataset 中记录的动作和实际演示不对应。

出现这些现象时，先停止程序，断电或复位，再重新确认端口和标签。
