# 100Hz 控制与数据质量

ZYArm 支持高频控制和状态读取方向，适合强化学习、模仿学习、VLA 和大模型控制实验。但 100Hz 不是一个简单的“数据集帧率”标签，需要拆开理解。

## 先区分几种频率

| 项目 | 含义 |
| --- | --- |
| 固件/SDK 控制频率 | 程序向机械臂下发动作的频率 |
| 状态读取频率 | 程序读取或缓存机械臂状态的频率 |
| `teleop.leader_hz` | LeRobot leader 动作读取频率，当前默认值为 `50.0` |
| camera fps | 摄像头图像采集帧率，例如 `30fps` |
| display 刷新 | `--display_data=true` 时界面显示数据的频率 |
| 视频编码速度 | 采集过程中把图像写成视频的处理速度 |
| dataset 写入速度 | observation、action、图像和元数据落盘速度 |

其中任何一环卡住，都会影响端到端采集体验。

## SDK/固件高频能力

实际使用时常见路径包括：

| 能力 | 含义 |
| --- | --- |
| `query_state()` | 主动向机械臂查询状态 |
| `get_latest_state()` | 读取 SDK 缓存的最新状态 |
| `fast_io()` | 高频动作下发路径 |
| `CMD36 fast_io` | 固件侧高频输入输出相关指令 |

这些能力适合程序循环和算法控制，不建议用串口软件手工模拟。

## LeRobot 中的相关配置

常见配置包括：

| 字段 | 默认值 | 说明 |
| --- | --- | --- |
| `robot.state_max_age_ms` | `100.0` | observation 状态缓存最大年龄 |
| `robot.slave_filter_lpf_alpha` | `0.15` | follower slave filter 的低通滤波系数 |
| `teleop.leader_hz` | `50.0` | leader 动作读取频率 |
| camera `fps` | 示例为 `30` | 相机目标采集帧率 |
| `--dataset.encoder_threads` | 示例为 `2` | 视频编码线程数 |

`robot.slave_filter_lpf_alpha` 会影响 follower 的平滑和响应。参数调整会影响手感、延迟和数据质量，属于需要谨慎验证的配置。

## 数据质量检查

采集后建议检查：

- 动作和画面是否同步。
- 夹爪和物体是否在关键视角内。
- episode 是否包含完整任务开始和结束。
- 是否有长时间静止、丢帧或卡顿。
- replay 是否能复现动作。
- 任务描述是否能让后续训练或评估理解当前任务。
- 端口、相机和物理标签是否与记录一致。

## 优化顺序

如果采集质量不好，建议按下面顺序排查：

1. 先关闭 `--display_data`，确认显示是否拖慢流程。
2. 降低相机分辨率。
3. 降低相机 fps。
4. 减少相机数量。
5. 调整 `encoder_threads`。
6. 缩短 episode，先做短采集和短 replay。
7. 再考虑调整 leader/follower 控制频率或滤波参数。

## 重要边界

- 100Hz 更适合描述控制/状态链路能力，不要把它直接等同于相机帧率。
- 摄像头 30fps 并不妨碍机械臂控制链路以更高频率运行。
- 数据集质量不只看频率，也看任务完整性、视角正确性、动作稳定性和 replay 可复现性。
- 高频控制参数修改如果涉及插件源码或 SDK 行为，请进入 [LeRobot 开发](../09_开发者指南/07_LeRobot开发/README.md) 和 [SDK 开发](../09_开发者指南/04_SDK开发/README.md)。
