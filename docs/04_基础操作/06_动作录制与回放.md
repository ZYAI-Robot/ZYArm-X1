# 动作录制与回放

固件支持按名称录制和回放动作。动作回放会让机械臂重复运动，第一次使用必须空载、低风险、周围无障碍，并确认可以随时断电。

动作录制可以理解成“把每个时刻的关节角度和夹爪值保存下来”；动作回放则是让固件按保存的帧序列重新执行。它不是视频录制，也不会保存摄像头画面。

## 推荐流程

```text
[CMD][1]
[CMD][13][demo_01]
...等待 5 秒倒计结束，机械臂卸力并开始录制...
...轻柔拖动机械臂完成动作...
[CMD][15]
[CMD][16]
[CMD][20][demo_01]
[CMD][14][demo_01]
[CMD][1]
```

录制动作适合做教学演示和简单复现。录制开始后，固件会先倒计 5 秒，再让关节卸力，方便手动掰动机械臂进行示教。不要在机械臂尚未卸力前强行掰动关节，否则可能导致机械臂损坏。

## CMD13 record

用途：按名称开始录制当前运动。

```text
[CMD][13][record_name]
```

示例：

```text
[CMD][13][pick_demo]
```

使用注意：

- `record_name` 建议使用英文、数字和下划线。
- 录制是持续过程，不是普通瞬时命令。
- 开始录制前建议先 `[CMD][1]` 复位，并确保动作空间安全。
- 发送指令后等待 5 秒倒计结束；看到固件提示开始录制后，再轻柔拖动机械臂。
- 录制开始时关节会卸力，录制结束后固件会重新锁定关节并更新当前角度。
- 不要在倒计期间或关节尚未卸力时强行掰动机械臂。

`record_name` 是动作名称，不是文件路径。多人共用设备时，建议使用能看懂的英文名，例如 `course01_pick`。

## CMD15 record_stop

用途：停止当前录制。

```text
[CMD][15]
```

使用注意：

- 停止录制后，再用 `CMD16` 查看是否已经生成记录。
- 如果录制过程中出现异常，优先断电处理，不要继续回放该动作。

## CMD16 record_list

用途：列出已录制动作。

```text
[CMD][16]
```

使用注意：

- 回放前先确认目标动作名存在。
- 多人共用设备时，建议为动作名加上课程、任务或日期前缀。

## CMD20 record_show

用途：打印指定录制动作的信息和每一帧关节角度。

```text
[CMD][20][record_name]
```

示例：

```text
[CMD][20][pick_demo]
```

使用注意：

- `CMD20` 只会把录制动作每个时刻的 `J0..J5` 和 `CLAW` 角度值打印出来，不会运动机械臂。
- 回放陌生动作前，先用 `CMD20` 查看帧数据和动作长度。
- 如果动作信息不清楚，不建议直接回放。

这里的“帧”表示某一个采样时刻的关节状态。帧越多，动作越长，占用的录制空间也越多。

## CMD14 record_player

用途：按名称回放已录制动作。

```text
[CMD][14][record_name]
```

示例：

```text
[CMD][14][pick_demo]
```

使用注意：

- 首次回放必须空载。
- 固件会先让机械臂运动到录制动作的第一帧状态，然后再开始播放完整轨迹。
- 回放过程中保持手远离机械臂和夹爪。
- 回放前仍需确认第一帧姿态附近没有障碍物，避免机械臂移动到起始帧时发生碰撞。

## CMD19 record_clear

用途：删除指定录制动作。

```text
[CMD][19][record_name]
```

删除所有录制动作：

```text
[CMD][19][all]
```

使用注意：

- 删除前先用 `CMD16` 和 `CMD20` 确认动作名。
- 录制动作数量有上限，当前固件最多保存 3 个动作；空间满后需要先删除旧动作，才能继续录制。
- `all` 会删除所有录制动作，删除后可能无法恢复，课程或比赛现场应谨慎操作。

## 容量和时长

当前固件配置来自 [firmware/Core/Inc/arm_recorder.h](../../firmware/Core/Inc/arm_recorder.h)：

- 最多保存 3 个动作。
- 每个动作最多记录 3000 帧。
- 默认记录间隔为 100ms。
- 单个动作最长约 300 秒。

空间满时，`CMD13` 会失败并提示需要清理 Flash。可以用 `CMD19` 删除单个动作，也可以用 `[CMD][19][all]` 删除全部动作。
