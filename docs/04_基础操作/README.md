# 基础操作

本章面向已经完成 [快速上手](../02_快速上手/README.md) 的用户，聚焦机械臂固件直接支持的串口指令。这里讲的是最底层、最容易验证的控制能力：你用串口软件发送 `[CMD][id][params]`，固件执行并返回状态或 ACK。

本章不展开 SDK、Web、ROS 2、MoveIt、Gazebo 或 LeRobot 的高级用法。等你理解这些底层指令后，再进入 [仿真与框架接入](../06_仿真与框架接入/README.md) 或 [科研与数据采集](../07_科研与数据采集/README.md) 会更稳。

## 先认识这些词

| 术语 | 本章里的含义 |
| --- | --- |
| 固件指令 | 直接发给控制板固件的文本命令，例如 `[CMD][6]` |
| `ACK` | 固件收到、返回或完成命令时输出的确认信息 |
| `STATUS` | 机械臂当前关节和夹爪状态 |
| IK | 运动学逆解，根据末端目标位姿计算关节角度 |
| 录制 / 回放 | 记录一段动作，再按名称播放 |
| 主从模式 | 一台机械臂作为 master，另一台作为 slave 跟随 |

更完整的名词说明见 [术语与工具速查](../术语与工具速查.md)。

## 指令来源

命令 ID 以 [firmware/Core/Inc/arm_shell.h](../../firmware/Core/Inc/arm_shell.h) 的 `CMD_ID_*` 枚举为准，命令名称和帮助信息以 [firmware/Core/Src/arm_shell_cmd.c](../../firmware/Core/Src/arm_shell_cmd.c) 的 `g_shell_cmd_list` 为准。

> 注意：不要按连续编号猜测命令。当前 `CMD31` 在枚举中存在，但没有注册到 `g_shell_cmd_list`，不作为对外使用指令。
