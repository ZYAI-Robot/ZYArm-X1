# 串口指令格式与回包

固件串口指令使用统一格式：

```text
[CMD][命令ID][参数]
```

`[CMD]` 表示这是一条发给固件的命令，`命令ID` 是固件中注册的数字编号，`参数` 是这条命令需要的输入。没有参数的命令也要保留前两段格式。

没有参数时可以只发送前两段：

```text
[CMD][6]
```

串口参数沿用快速上手中的设置：`230400, 8N1, no flow control`。如果你还没有完成串口连接，请先阅读 [快速上手](../02_快速上手/README.md)。

## 浮点参数指令

大多数控制指令使用浮点参数，参数之间用空格分隔：

```text
[CMD][0][200 0 0 0 0 0]
[CMD][5][0 10]
[CMD][3][0 20 30 0 0 0 0]
```

固件最多解析 10 个浮点参数。参数数量不足时，固件会输出错误日志，并对已经识别出的命令返回 `ACK_COMPLETED: ERROR`。

“浮点参数”就是可以带小数的数字，例如角度、位置、频率或阈值。手工输入时请用英文空格分隔，不要使用中文逗号。

## 字符串参数指令

少量指令使用字符串参数，例如设置机械臂名称、录制动作名称：

```text
[CMD][21][master]
[CMD][13][pick_demo]
```

字符串最大长度来自 `ARM_NAME_MAX_LEN`，当前为 32。建议只使用英文、数字和下划线，避免不同串口软件对中文或特殊字符编码处理不一致。

“字符串参数”就是一段名称文本，例如 `master` 或 `pick_demo`。它通常用于设备命名和动作命名，不表示关节角度。

## 常见回包

固件通常会返回三类信息：

```text
ACK_RECEIVED: CMD_ID=6
ACK_RESPONSE: CMD_ID=6, [STATUS] J0:0.00 J1:0.00 J2:0.00 J3:0.00 J4:0.00 J5:0.00 CLAW:0.00
ACK_COMPLETED: CMD_ID=6, SUCCESS
```

含义如下：

| 回包 | 含义 |
| --- | --- |
| `ACK_RECEIVED` | 固件已收到并开始处理该指令 |
| `ACK_RESPONSE` | 指令产生了数据返回，例如状态、版本、名称或诊断统计 |
| `ACK_COMPLETED: SUCCESS` | 指令执行成功 |
| `ACK_COMPLETED: ERROR` | 指令执行失败，通常需要查看前后的错误日志 |

`CMD2 stop` 是特殊急停指令，会在解析任务中直接处理；固件仍会返回 `ACK_RECEIVED` 和 `ACK_COMPLETED`，方便上位机判断命令生命周期已经结束。

## help

串口中发送：

```text
help
```

固件会打印当前注册的命令列表。命令列表来自 `g_shell_cmd_list`，适合用来确认当前固件实际支持哪些命令。

## 使用建议

- 每次动作前先发送 `[CMD][6]` 读取状态。
- 首次尝试动作时优先使用 `[CMD][1]` 复位、`[CMD][0]` 小范围 IK 或 `[CMD][5]` 单关节小角度动作。
- 遇到异常运动时优先直接切断机械臂电源；串口已连通且来得及发送时，可发送 `[CMD][2]` 指令急停。
- 如果串口软件看不到回包，先检查串口号、波特率、流控、USB 线和控制板供电。
