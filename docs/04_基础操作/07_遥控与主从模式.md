# 遥控与主从模式

本页整理遥控和主从跟随相关固件指令。这些能力通常需要更完整的设备准备，例如双机械臂、稳定供电、线材固定和角色标记。接线和设备组织请阅读 [主从臂遥操](../05_常用玩法/04_主从臂遥操.md)。

遥控模式用于连续根据外部输入更新机械臂目标；主从模式用于让一台机械臂作为 master，另一台作为 slave 跟随。它们都比单次 `[CMD]` 动作复杂，推荐通过脚本使用。

## 推荐入口：脚本优先

遥控和主从跟随不建议直接在串口软件里手工连续发送底层固件命令。实际使用时，优先运行仓库提供的脚本或上层程序，让程序处理初始化、频率、ACK、状态同步和退出清理。

| 场景 | 推荐入口 | 说明 |
| --- | --- | --- |
| Joy-Con / 手柄遥控 | [arm_serial_controller.py](../../software/tools/arm_serial_controller.py) | 自动进入遥控模式，执行遥控复位，并连续发送 `CMD26` |
| fast_io 主从遥操 | [fast_io_teleop_pair.py](../../software/tools/fast_io_teleop_pair.py) | 基于 SDK 的高频主从遥操工具 |
| 固件主从跟随 | [filtered_master_slave_teleop.py](../../software/tools/filtered_master_slave_teleop.py) | 负责启动 master/slave、日志和停止流程 |
| 旧主从入口 | [master_slave_remote.py](../../software/tools/master_slave_remote.py) | 兼容 wrapper，后续优先使用 `filtered_master_slave_teleop.py` |

下面的 `CMD24`、`CMD25`、`CMD26`、`CMD32`、`CMD33` 和 `CMD34` 主要作为底层指令参考，帮助你理解脚本背后调用了什么。

表中的脚本会帮你处理端口打开、发送频率、复位顺序和退出清理。手工连续发送遥控命令很容易漏掉这些步骤。

## CMD24 remote_mode

用途：开启或关闭遥控模式。

```text
[CMD][24][1]
[CMD][24][0]
```

参数说明：

| 参数 | 含义 |
| --- | --- |
| `0` | 关闭遥控模式，回到空闲状态 |
| `1` | 开启遥控模式 |

使用注意：

- 遥控模式不是首次上手必需能力。
- 进入遥控模式后，通常需要配合 `CMD25` 和 `CMD26` 使用。
- 手柄遥控建议通过 [arm_serial_controller.py](../../software/tools/arm_serial_controller.py) 使用，不建议手工发送连续遥控命令。

## CMD25 remote_reset

用途：在遥控模式下复位到初始姿态。

```text
[CMD][25]
```

使用注意：

- 该指令只能在遥控模式下使用。
- 遥控开始前建议先复位，保证后续姿态基准一致。

## CMD26 remote

用途：遥控末端位姿和夹爪角度。

```text
[CMD][26][x y z rx ry rz claw_angle]
```

参数说明：

| 参数 | 含义 |
| --- | --- |
| `x y z` | 末端目标位置 |
| `rx ry rz` | 末端目标姿态 |
| `claw_angle` | 夹爪角度 |

使用注意：

- 该指令只有在遥控模式下生效。
- 固件中会检查遥控复位状态，未完成遥控复位时可能不会执行。
- 遥控指令适合连续控制，不建议新手直接手工高频发送。
- 手工调试时只适合低频、少量验证；真实遥控体验请使用脚本。

## CMD32 master_slave

用途：启动主从跟随模式。

```text
[CMD][32][1]
[CMD][32][2]
```

可选追加频率：

```text
[CMD][32][1 50]
[CMD][32][2 50]
```

参数说明：

| 参数 | 含义 |
| --- | --- |
| `role` | `1` 为 master，`2` 为 slave |
| `freq_hz` | 可选，跟随频率，默认 50Hz |

`freq_hz` 表示每秒发送或处理多少次跟随数据。频率越高不一定越稳，还要看串口、供电、脚本和机械负载是否能跟上。

使用注意：

- 启动前先确认两台机械臂名称、端口和物理角色没有接反。
- 两台机械臂都应能单独完成 `CMD6` 状态读取和 `CMD1` 复位。
- 多设备连接和命名见 [主从臂遥操](../05_常用玩法/04_主从臂遥操.md)。
- 主从遥操建议通过 [filtered_master_slave_teleop.py](../../software/tools/filtered_master_slave_teleop.py) 或 [fast_io_teleop_pair.py](../../software/tools/fast_io_teleop_pair.py) 启动，不建议手工分别给两台机械臂发送启动命令。

## CMD33 master_slave_stop

用途：停止主从跟随模式。

```text
[CMD][33]
```

使用注意：

- 停止后仍应检查两台机械臂状态。
- 如果主从运动异常，优先断电处理。

## CMD34 master_slave_set_lpf

用途：设置主从跟随低通滤波系数。

```text
[CMD][34][alpha]
```

参数说明：

| 参数 | 含义 |
| --- | --- |
| `alpha` | 低通滤波系数，固件帮助信息标注为 `0.0-1.0` |

使用注意：

- 该参数会影响跟随手感和响应。
- 不建议在未理解主从链路前随意修改。
- 如果通过脚本运行主从遥操，优先使用脚本暴露的参数，不要同时手工修改固件 LPF。
