# 状态查询与版本信息

本页整理读取状态、版本、速度、名称和单关节信息的指令。这些指令风险低，适合在每次动作前后反复使用。

## CMD6 status

用途：读取 6 个关节和夹爪的当前状态。

```text
[CMD][6]
```

可选参数：

```text
[CMD][6][1]
```

`verbose` 不为 0 时，固件会额外查询舵机状态。首次上手建议先使用无参数版本。

`verbose` 可以理解成“输出更多信息”的开关。它适合排障，不适合第一次上手时默认开启。

典型返回：

```text
ACK_RESPONSE: CMD_ID=6, [STATUS] J0:0.00 J1:0.00 J2:0.00 J3:0.00 J4:0.00 J5:0.00 CLAW:0.00
```

使用注意：

- `J0` 到 `J5` 对应 6 个关节角度。
- `CLAW` 对应夹爪角度。
- 快速上手中的“记录状态并复现”就是先用 `CMD6` 记录角度，再用 `CMD3` 回放。

## CMD4 version

用途：读取硬件版本、软件版本和构建时间。

```text
[CMD][4]
```

典型返回包含：

```text
Hardware Version: ...
Software Version: ...
Build Date: ...
```

使用注意：

- 提交 issue、排查固件差异或确认烧录版本时，建议先记录 `CMD4` 输出。
- 多台机械臂混用时，可以配合 `CMD22` 一起确认设备身份。

## CMD22 get_name

用途：读取机械臂名称。

```text
[CMD][22]
```

典型返回：

```text
ACK_RESPONSE: CMD_ID=22, NAME:master
```

使用注意：

- 双臂或多臂场景建议给不同机械臂设置不同名称。
- 名称写入方式见 `CMD21 set_name`。
- 双臂设备记录见 [主从臂遥操](../05_常用玩法/04_主从臂遥操.md)，LeRobot 采集设备记录见 [设备角色与端口确认](../07_科研与数据采集/03_设备角色与端口确认.md)。

## CMD21 set_name

用途：设置机械臂名称。

```text
[CMD][21][master]
[CMD][21][slave]
```

参数说明：

| 参数 | 含义 |
| --- | --- |
| `name` | 机械臂名称，建议使用英文、数字和下划线 |

使用注意：

- 名称会保存到 Flash。
- 不建议频繁反复写入名称。
- 设置后用 `[CMD][22]` 确认。

Flash 是控制板上的非易失存储，断电后仍能保存名称等配置。它适合保存少量配置，但不适合被高频反复写入。

## CMD10 get_joint

用途：读取指定关节角度。

```text
[CMD][10][0]
```

参数说明：

| 参数 | 含义 |
| --- | --- |
| `joint_id` | 关节编号，通常 `0..5` 为机械臂关节，夹爪使用专用指令读取 |

典型返回：

```text
ACK_RESPONSE: CMD_ID=10, J0:0.00
```

使用注意：

- 如果只是想一次看全部状态，优先使用 `CMD6`。
- 单独排查某个关节时再使用 `CMD10`。

## CMD9 get_claw

用途：读取当前夹爪位置。

```text
[CMD][9]
```

典型返回：

```text
ACK_RESPONSE: CMD_ID=9, CLAW:0.00
```

使用注意：

- 夹爪位置单位沿用固件内部角度表达。
- 上层 SDK 或 LeRobot 可能会使用归一化表达，本章不展开这些映射。

## CMD12 get_speed

用途：读取当前关节速度配置。

```text
[CMD][12]
```

典型返回：

```text
ACK_RESPONSE: CMD_ID=12, SPEED:...
```

使用注意：

- 速度单位为 `degree/s`。
- 快速上手流程不要求设置速度，默认速度即可。

`degree/s` 表示每秒转过多少角度。速度越高，动作越快，风险也越高。
