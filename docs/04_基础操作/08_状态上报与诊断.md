# 状态上报与诊断

本页整理周期状态上报和串口通信诊断指令。这些能力常用于持续监控、脚本控制、数据采集和排障。

周期状态上报是“固件主动按频率持续输出状态”；诊断指令是“查看通信链路有没有错误、超时或丢包”。第一次上手一般不需要开启周期上报。

## CMD17 status_report

用途：开启或关闭周期状态上报。

```text
[CMD][17][1]
[CMD][17][0]
```

可选指定频率：

```text
[CMD][17][1 20]
[CMD][17][1 100]
```

参数说明：

| 参数 | 含义 |
| --- | --- |
| `enable` | `1` 开启，`0` 关闭 |
| `freq_hz` | 可选，上报频率，默认 20Hz |

`freq_hz` 表示每秒上报多少次状态，例如 `20` 表示约每秒 20 次，`100` 表示约每秒 100 次。频率越高，对上位机读取能力要求越高。

使用注意：

- 周期上报会持续占用串口输出。
- 使用串口软件手工调试时，建议先低频开启。
- 做 100Hz 控制或数据采集前，应确认上位机能稳定读取串口。

## CMD18 get_status_report

用途：查询周期状态上报是否开启以及当前频率。

```text
[CMD][18]
```

典型返回：

```text
ACK_RESPONSE: CMD_ID=18, STATUS_REPORT:RUNNING=1,FREQ=20
```

使用注意：

- 如果串口一直刷状态，可以先用 `CMD18` 确认是否开启了周期上报。
- 关闭周期上报使用 `[CMD][17][0]`。

## CMD35 get_transport_stats

用途：读取 UART1、命令解析器和 UART6 舵机通信健康快照。

```text
[CMD][35]
```

典型返回包含：

```text
TRANSPORT_UART1_STATS:...
CMD_PARSER_STATS:...
SERVO_UART6_STATS:...
SERVO_REQ_REPLY:...
```

使用注意：

- 该指令适合排查串口丢包、解析错误、舵机通信超时和链路稳定性。
- 提交问题时，建议同时附上 `CMD4`、`CMD6` 和 `CMD35` 输出。
- 初学者日常操作不需要频繁使用。

这里的 UART 是控制板上的串口外设：UART1 面向上位机命令链路，UART6 面向舵机通信链路。诊断输出用于定位是哪一段通信不稳定。

## 和 CMD36 的区别

`CMD17` 是周期状态上报，适合只读监控。`CMD36 joint_io_fast` 是一次指令里同时做关节目标下发和状态输出，适合高频控制链路。二者都可能产生较高频串口输出，使用前要确认上位机接收能力。

## 排障建议

- 看不到任何回包：先检查串口号、波特率、供电、USB 线和流控设置。
- 有 `CMD_PARSER_STATS:ERROR` 增长：检查指令格式、参数数量和换行发送方式。
- 有舵机超时统计增长：检查机械臂供电、舵机线材、负载和是否发生碰撞。
- 高频状态丢失：降低 `freq_hz`，或改用稳定脚本读取串口。
