# 关节与夹爪控制

本页整理关节、夹爪、速度和快速关节 IO 指令。初次练习建议从 `CMD5` 单关节小角度动作开始，再使用 `CMD3` 同步控制多个关节。

“关节角度”表示某个关节当前或目标转到的位置；“夹爪角度”表示夹爪开合位置。本页直接使用固件角度表达，和 SDK/ROS 中的用户单位可能不同。

## 关节编号

常用编号：

| 编号 | 含义 |
| ---: | --- |
| `0..5` | 机械臂 6 个关节 |
| `CLAW` | 夹爪，通常通过 `CMD8` / `CMD9` 或 `CMD3` 的第 7 个参数控制 |

具体关节方向、机械零位和安全角度范围需要以项目方实物资料为准。

> 待项目方补充：请提供 J0 到 J5 的实物标注图、正方向示意图和推荐安全角度范围。

## CMD5 set_joint

用途：设置单个关节角度。

```text
[CMD][5][joint_id angle]
```

也可以追加 `sync` 参数：

```text
[CMD][5][0 10 1]
```

参数说明：

| 参数 | 含义 |
| --- | --- |
| `joint_id` | 关节编号 |
| `angle` | 目标角度 |
| `sync` | 可选，是否等待动作完成，默认 `1` |

`sync` 可以理解成“这条命令是否等待动作执行完成再返回完成信息”。首次手工调试使用默认值即可。

入门示例：

```text
[CMD][6]
[CMD][5][0 10]
[CMD][6]
[CMD][1]
```

使用注意：

- 首次测试只让一个关节小角度运动。
- 不清楚安全角度范围时，不要发送大角度。
- 如果只想验证整机链路，快速上手中的 `CMD0` IK 小动作通常更简单。

## CMD3 joint_sync

用途：同步设置 6 个关节和夹爪。

```text
[CMD][3][j0 j1 j2 j3 j4 j5 claw]
```

可选完整格式：

```text
[CMD][3][j0 j1 j2 j3 j4 j5 claw sync verbose fast]
```

参数说明：

| 参数 | 含义 |
| --- | --- |
| `j0..j5` | 6 个关节目标角度 |
| `claw` | 夹爪目标角度 |
| `sync` | 可选，是否等待同步完成，默认 `1` |
| `verbose` | 可选，非 0 时输出目标正解位姿 |
| `fast` | 可选，非 0 时走快速路径 |

`verbose` 用于输出更多计算信息，`fast` 用于切换快速路径。首次练习普通 `CMD3` 时不需要填写这两个参数。

保持某些关节不变：

```text
[CMD][3][-999.9 20 -999.9 -999.9 -999.9 -999.9 -999.9]
```

`-999.9` 来自固件中的 `ARM_SYNC_NO_CHANGE`，表示该关节保持当前角度。

这个特殊值不是目标角度，而是“这个关节不改变”的占位符。只有在你明确想保持某些关节不动时才使用。

使用注意：

- `CMD3` 会直接下发多个目标角度，比 `CMD5` 更容易造成大范围动作。
- 建议先通过 `[CMD][6]` 记录当前状态，再只改一个或两个参数。
- 快速上手里的“记录状态并复现”使用的就是 `CMD3`。

## CMD8 set_claw

用途：设置夹爪位置。

```text
[CMD][8][position]
```

可选追加 `sync`：

```text
[CMD][8][20 1]
```

参数说明：

| 参数 | 含义 |
| --- | --- |
| `position` | 夹爪目标角度 |
| `sync` | 可选，是否等待动作完成，默认 `1` |

使用注意：

- 首次测试时不要让夹爪夹住硬物。
- 夹爪附近不要放手指、线材或易碎物。
- 当前文档不假设具体开合角度范围，项目方补充实物规格后再完善。

## CMD11 set_speed

用途：设置关节速度，单位为 `degree/s`。

```text
[CMD][11][speed]
```

示例：

```text
[CMD][11][30]
[CMD][12]
```

使用注意：

- 速度配置会保存到 Flash。
- 快速上手不要求设置速度，默认速度即可。
- 不建议学习者随意设置很高速度。

> 待项目方补充：请提供入门教学、课程实验、比赛任务和不同负载下的推荐速度范围。

## CMD36 joint_io_fast

用途：批量读取关节快照、快速同步下发目标，并输出 `[STATUS]`。

```text
[CMD][36][j0 j1 j2 j3 j4 j5 claw]
```

使用注意：

- 这是高频控制和数据采集更常用的底层能力。
- 该指令更适合已经理解 `CMD3`、状态回包和机械臂安全边界的用户。
- 新手做单次动作验证时，优先使用 `CMD0`、`CMD5` 或普通 `CMD3`。

这里的 IO 指 input/output：一条命令同时包含动作输入和状态输出。它适合程序循环调用，不适合新手手工快速连发。
